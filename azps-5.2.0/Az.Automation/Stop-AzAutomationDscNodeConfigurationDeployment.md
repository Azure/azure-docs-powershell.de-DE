# Stop-AzAutomationDscNodeConfigurationDeployment

## SYNOPSIS
Beendet eine DSC-Knoten-Konfigurationsbereitstellung in der Automatisierung. Der aktuelle Bereitstellungsauftrag wird nur beendet, die zugewiesenen Knotenkonfigurationen werden jedoch nicht entfernt.

## SYNTAX

### ByJobId (Standard)
```
Stop-AzAutomationDscNodeConfigurationDeployment -JobId <Guid> [-Force] [-PassThru]
 [-ResourceGroupName] <String> [-AutomationAccountName] <String> [-DefaultProfile <IAzureContextContainer>]
 [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ByInputObject
```
Stop-AzAutomationDscNodeConfigurationDeployment [-PassThru] -InputObject <NodeConfigurationDeployment>
 [-ResourceGroupName] <String> [-AutomationAccountName] <String> [-DefaultProfile <IAzureContextContainer>]
 [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESCHREIBUNG
Das **Cmdlet "Stop-AzAutomationDscNodeConfigurationDeployment"** beendet die Bereitstellung einer Konfiguration des Knotens "Desired State Configuration (DSC)" in der Azure-Automatisierung. Damit wird die Zuweisung der Knotenkonfiguration zu Knotengruppen beendet, sofern noch Knoten zugewiesen werden müssen, die Zuweisung bereits zugewiesener Knoten wird jedoch nicht beendet. Wenn Sie die Registrierung eines geplanten Auftrags aufheben möchten, verwenden Sie das ["Registrierung aufheben"-AzAutomationScheduledRunbook](./Unregister-AzAutomationScheduledRunbook.md) zusammen mit "JobScheduleId", um die Zuzuweisen eines vorhandenen geplanten Auftrags auf aufheben.

## BEISPIELE

### Beispiel 1: Bereitstellen einer Azure DSC-Knotenkonfiguration in der Automatisierung
```
PS C:\> Stop-AzAutomationDscNodeConfigurationDeployment -AutomationAccountName "Contoso01" -ResourceGroupName "ResourceGroup01" -JobId 00000000-0000-0000-0000-000000000000
```

Der oben aufgeführte Befehl beendet den Bereitstellungsauftrag der DSC-Knotenkonfiguration mit der übergebenen jobId.

# Reset-AzHubRouter

## SYNOPSIS
Setzt das RoutingState einer VirtualHub-Ressource zurück.

## SYNTAX

### ByVirtualHubName (Standard)
```
Reset-AzHubRouter -ResourceGroupName <String> -Name <String> [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ByVirtualHubResourceId
```
Reset-AzHubRouter -ResourceId <String> [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ByVirtualHubObject
```
Reset-AzHubRouter -InputObject <PSVirtualHub> [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESCHREIBUNG
Setzt den Routingstatus einer vorhandenen VirtualHub-Ressource nur zurück, wenn der Routingstatus des virtuellen Hubs nicht bereitgestellt wird.

## BEISPIELE

### Beispiel 1

```powershell
PS C:\> Reset-AzHubRouter -ResourceGroupName "testRG" -Name "westushub"
```

Setzen Sie den Routingstatus des virtuellen Hubs mithilfe von ResourceGroupName und ResourceName zurück.

### Beispiel 2

```powershell
PS C:\> Reset-AzHubRouter -ResourceId "/subscriptions/testSub/resourceGroups/testRG/providers/Microsoft.Network/virtualHubs/westushub"
```

Setzen Sie den Routingstatus des virtuellen Hubs mit seiner ResourceId zurück.

### Beispiel 3

```powershell
PS C:\> Reset-AzHubRouter -InputObject $virtualHub
```

Setzen Sie den Routingstatus des virtuellen Hubs mithilfe eines Eingabeobjekts zurück. Das Eingabeobjekt ist vom Typ "PSVirtualHub".

### Beispiel 4

```powershell
PS C:\> Get-AzVirtualHub -ResourceGroupName "testRG" -Name "westushub" | Reset-AzHubRouter
```

Ein vorhandenes virtuelles Hubobjekt kann abgerufen und dann als Eingabeobjekt an Reset-AzHubRouter übergeben werden.

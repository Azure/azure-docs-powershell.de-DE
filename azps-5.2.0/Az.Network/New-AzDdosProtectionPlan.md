# New-AzDdosProtectionPlan

## SYNOPSIS
Erstellt einen DDoS-Schutzplan.

## SYNTAX

```
New-AzDdosProtectionPlan -ResourceGroupName <String> -Name <String> -Location <String> [-Tag <Hashtable>]
 [-AsJob] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESCHREIBUNG
Das New-AzDdosProtectionPlan erstellt einen DDoS-Schutzplan.

## BEISPIELE

### Beispiel 1: Erstellen und Zuordnen eines DDoS-Schutzplans zu einem neuen virtuellen Netzwerk
```
D:\> $ddosProtectionPlan = New-AzDdosProtectionPlan -ResourceGroupName ResourceGroupName -Name DdosProtectionPlanName -Location "West US"
D:\> $subnet = New-AzVirtualNetworkSubnetConfig -Name SubnetName -AddressPrefix 10.0.1.0/24
D:\> $vnet = New-AzVirtualNetwork -Name VnetName -ResourceGroupName ResourceGroupName -Location "West US" -AddressPrefix 10.0.0.0/16 -DnsServer 8.8.8.8 -Subnet $subnet -EnableDdoSProtection -DdosProtectionPlanId $ddosProtectionPlan.Id
```

Zuerst erstellen wir einen neuen DDoS Protection-Plan mit dem **Befehl "New-AzDdosProtectionPlan".**
Anschließend erstellen wir ein neues virtuelles Netzwerk mit **New-AzVirtualNetwork** und geben die ID des neu erstellten Plans im Parameter **DdosProtectionPlanId an.** In diesem Fall können wir, da wir das virtuelle Netzwerk einem Plan zuordnen, auch den Parameter **"EnableDdoSProtection" angeben.**

### Beispiel 2: Erstellen und Zuordnen eines DDoS-Schutzplans zu einem vorhandenen virtuellen Netzwerk
```
D:\> $ddosProtectionPlan = New-AzDdosProtectionPlan -ResourceGroupName ResourceGroupName -Name DdosProtectionPlanName -Location "West US"
D:\> $vnet = Get-AzVirtualNetwork -Name VnetName -ResourceGroupName ResourceGroupName
D:\> $vnet.DdosProtectionPlan = New-Object Microsoft.Azure.Commands.Network.Models.PSResourceId
D:\> $vnet.DdosProtectionPlan.Id = $ddosProtectionPlan.Id
```

Zweitens erhalten wir die neueste Version des virtuellen Netzwerks, das wir dem Plan zuordnen möchten. Wir aktualisieren die Eigenschaft **"DdosProtectionPlan"** mit einem **PSResourceId-Objekt,** das einen Verweis auf die ID des neu erstellten Plans enthält. In diesem Fall können wir, wenn wir das virtuelle Netzwerk einem DDoS-Schutzplan zuordnen, auch die Kennzeichnung **"EnableDdosProtection"** auf "true" festlegen.
Schließlich speichern wir den neuen Zustand, indem wir die lokale Variable in **Set-AzVirtualNetwork verwenden.**

# Set-AzVMBootDiagnostic

## SYNOPSIS
Ändert die Eigenschaften der Startdiagnose eines virtuellen Computers.

## SYNTAX

### EnableBootDiagnostics
```
Set-AzVMBootDiagnostic [-VM] <PSVirtualMachine> [-Enable] [-ResourceGroupName] <String>
 [[-StorageAccountName] <String>] [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

### DisableBootDiagnostics
```
Set-AzVMBootDiagnostic [-VM] <PSVirtualMachine> [-Disable] [-DefaultProfile <IAzureContextContainer>]
 [<CommonParameters>]
```

## BESCHREIBUNG
Das **Cmdlet "Set-AzVMBootDiagnostic"** ändert die Eigenschaften der Startdiagnose eines virtuellen Computers.

## BEISPIELE

### Beispiel 1: Aktivieren der Startdiagnose
```
PS C:\> $VM = Get-AzVM -ResourceGroupName "ResourceGroup11" -Name "ContosoVM07"
PS C:\> Set-AzVMBootDiagnostic -VM $VM -Enable -ResourceGroupName "ResourceGroup11" -StorageAccountName "DiagnosticStorage"
PS C:\> Update-AzVM -VM $VM
```

Der erste Befehl ruft den virtuellen Computer "ContosoVM07" mit **"Get-AzVM" ab.**
Der Befehl speichert sie in der $VM Variable.
Der zweite Befehl ermöglicht die Startdiagnose für den virtuellen Computer in $VM.
Diagnosedaten werden im angegebenen Konto gespeichert.

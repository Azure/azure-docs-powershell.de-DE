# New-AzStreamAnalyticsTransformation

## SYNOPSIS
Erstellt oder aktualisiert eine Transformation innerhalb eines Auftrags.

## SYNTAX

```
New-AzStreamAnalyticsTransformation [-JobName] <String> [[-Name] <String>] [-File] <String> [-Force]
 [-ResourceGroupName] <String> [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## BESCHREIBUNG
Das **Cmdlet "New-AzStreamAnalyticsTransformation"** erstellt eine Transformation in einem Stream Analytics-Auftrag oder aktualisiert die vorhandene Transformation.
Der Name der Transformation kann im angegeben werden. JSON-Datei oder in der Befehlszeile.
Wenn beide Angegeben werden, muss der Name in der Befehlszeile mit dem Namen in der Datei übereinstimmen.
Wenn Sie eine Transformation angeben, die bereits vorhanden ist, ohne den Parameter "Force" anzugeben, fragt das Cmdlet, ob die vorhandene Transformation ersetzt werden soll.
Wenn Sie den Parameter *"Force"* und einen vorhandenen Transformationsnamen angeben, wird die Transformation ohne Bestätigung ersetzt.

## BEISPIELE

### Beispiel 1: Erstellen oder Ersetzen einer Transformation in einem Auftrag
```powershell
PS C:\>New-AzStreamAnalyticsTransformation -ResourceGroupName "StreamAnalytics-Default-West-US" -File "C:\Transformation.json" -JobName "StreamingJob" -Name "StreamingJobTransform"
```

Mit diesem Befehl wird eine Transformation namens "StreamingJobTransform" im Auftrag "StreamingJob" erstellt.
Wenn eine vorhandene Transformation bereits mit diesem Namen definiert ist, fragt das Cmdlet, ob sie ersetzt werden soll.

### Beispiel 2: Ersetzen einer Transformation in einem Auftrag
```powershell
PS C:\>New-AzStreamAnalyticsTransformation -ResourceGroupName "StreamAnalytics-Default-West-US" -File "C:\Transformation.json" -JobName "StreamingJob" -Name "StreamingJobTransform" -Force
```

Dieser Befehl ersetzt die Definition von StreamingJobTransform im Auftrag von StreamingJob ohne Bestätigung.

# Get-AzSupportService

## SYNOPSIS
Holen Sie sich Dienste, für die Support verfügbar ist. 

## SYNTAX

### ListParameterSet (Standard)
```
Get-AzSupportService [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

### GetByNameParameterSet
```
Get-AzSupportService -Id <String> [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

## BESCHREIBUNG
Ruft die aktuelle Liste der Azure-Dienste ab, für die Support verfügbar ist. Jeder Dienst kann Informationen zum Ressourcentyp des Azure-Ressourcen-Managers (ARM) enthalten. Ressourcentypen (Beispiel: "microsoft.compute/virtualmachines")) können hilfreich sein, um das richtige Produkt und die richtige ARM zu zuordnungen, wenn Sie ein Ticket für den technischen Support erstellen. Jeder Azure Service verfügt über einen eigenen Satz von Kategorien, die als Problemklassifizierung bezeichnet werden. Hier finden Sie die aktuelle Liste der Problemklassifizierung für einen Dienst mit "Get-AzSupportProblemClassification". Sie können die GUID für Dienst- und Problemklassifizierung verwenden, um ein neues Supportticket mit "New-AzSupportTicket" zu erstellen.

Verwenden Sie immer die dienst- und problemklassifizierungs-GUIDs, die programmgesteuert ermittelt wurden. Durch diese Vorgehensweise wird sichergestellt, dass Sie über die neuesten GUIDs für Dienst- und Problemklassifizierungen für die Erstellung von Supporttickets verfügen.

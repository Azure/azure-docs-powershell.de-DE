# Restart-AzServiceFabricManagedNodeType

## Synopsis
Starten Sie bestimmte Knoten aus dem Knotentyp neu.

## Syntax

```
Restart-AzServiceFabricManagedNodeType [-ResourceGroupName] <String> [-ClusterName] <String> [-Name] <String>
 -NodeName <String[]> [-ForceRestart] [-PassThru] [-AsJob] [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

## Beschreibung
Starten Sie bestimmte Knoten aus dem Knotentyp neu. Der Dienst Fabric-Knoten wird deaktiviert, bevor die VMs neu gestartet und wieder aktiviert werden können. Wenn dies für primäre Knotentypen erfolgt, kann es eine Weile dauern, da es möglicherweise nicht alle Knoten gleichzeitig neu startet. Use-ForceRestart erzwingen Sie den Vorgang selbst dann, wenn Service Fabric die Knoten nicht deaktivieren, aber mit Vorsicht verwenden kann, da dies zu Datenverlusten führen kann, wenn statusbehaftete Arbeitslasten auf dem Knoten ausgeführt werden.

## Beispiele

### Beispiel 1
```powershell
$rgName = "testRG"
$clusterName = "testCluster"
$NodeTypeName = "nt1"
Restart-AzServiceFabricManagedNodeType -ResourceGroupName $rgName -ClusterName $clusterName  -Name $NodeTypeName -NodeName nt1_0, nt1_3
```

Starten Sie Knoten 0 und 3 auf dem Knotentyp neu.

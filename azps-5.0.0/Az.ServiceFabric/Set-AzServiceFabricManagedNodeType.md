# Set-AzServiceFabricManagedNodeType

## Synopsis
Legt Knotentyp-Ressourceneigenschaften fest oder führt reimage-Aktionen für bestimmte NDES des Knotentyps mit dem Parameter-reimage aus.

## Syntax

### ByObj (Standard)
```
Set-AzServiceFabricManagedNodeType [-InputObject] <PSManagedNodeType> [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### WithParamsByName
```
Set-AzServiceFabricManagedNodeType [-ResourceGroupName] <String> [-ClusterName] <String> [-Name] <String>
 [-AsJob] [-InstanceCount <Int32>] [-ApplicationStartPort <Int32>] [-ApplicationEndPort <Int32>]
 [-EphemeralStartPort <Int32>] [-EphemeralEndPort <Int32>] [-Capacity <Hashtable>]

```

```

```

## Beschreibung
Legt Knotentyp-Ressourceneigenschaften fest oder führt reimage-Aktionen für bestimmte NDES des Knotentyps mit dem Parameter-reimage aus. Bei einem reimgae-Vorgang werden die Service Fabric-Knoten deaktiviert, bevor Sie die VMs erneut abbilden und Sie wieder aktivieren, sobald Sie wieder verfügbar sind. Wenn dies für primäre Knotentypen erfolgt, kann es eine Weile dauern, da es möglicherweise nicht alle Knoten zur gleichen Zeit umbildet. Use-ForceReimage erzwingen Sie den Vorgang selbst dann, wenn Service Fabric die Knoten nicht deaktivieren, aber mit Vorsicht verwenden kann, da dies zu Datenverlusten führen kann, wenn statusbehaftete Arbeitslasten auf dem Knoten ausgeführt werden.

## Beispiele

### Beispiel 1
```powershell
$rgName = "testRG"
$clusterName = "testCluster"
$NodeTypeName = "nt1"
Set-AzServiceFabricManagedNodeType -ResourceGroupName $rgName -ClusterName $clusterName -name $NodeTypeName -InstanceCount 6 -Verbose
```

Aktualisieren Sie die Anzahl der Instanzen des Knotentyps.

### Beispiel 2
```powershell
$rgName = "testRG"
$clusterName = "testCluster"
$NodeTypeName = "nt1"
Set-AzServiceFabricManagedNodeType -ResourceGroupName $rgName -ClusterName $clusterName -name $NodeTypeName -PlacementProperty @{NodeColor="Red";SomeProperty="6";} -Verbose
```

Aktualisieren der Platzierungs-properites des Knotentyps Dadurch werden ältere Placement-properites, falls vorhanden, überschrieben.

### Beispiel 3
```powershell
$rgName = "testRG"
$clusterName = "testCluster"
$NodeTypeName = "nt1"
Set-AzServiceFabricManagedNodeType -ResourceGroupName $rgName -ClusterName $clusterName  -Name $NodeTypeName -Reimage -NodeName nt1_0, nt1_3
```

Umbilden von Knoten 0 und 3 auf dem Knotentyp

### Beispiel 4
```powershell
$rgName = "testRG"
$clusterName = "testCluster"
$NodeTypeName = "nt1"
$nodeType = Get-AzServiceFabricManagedNodeType -ResourceGroupName $rgName -ClusterName $clusterName -Name $NodeTypeName

$nodeType.VmInstanceCount = 6
$nodeType | Set-AzServiceFabricManagedNodeType
```

Aktualisieren Sie die Anzahl der Instanzen des Knotentyps mit Piping.

# New-AzDeploymentManagerStep

## Synopsis
Erstellt einen Schritt.

## Syntax

### Warten (Standard)
```
New-AzDeploymentManagerStep -ResourceGroupName <String> -Name <String> -Location <String> -Duration <String>
 [-Tag <Hashtable>] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### HealthCheckFile
```
New-AzDeploymentManagerStep -ResourceGroupName <String> -Name <String> -Location <String>
 -HealthCheckPropertiesFile <String> [-Tag <Hashtable>] [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

### HealthCheckObject
```
New-AzDeploymentManagerStep -ResourceGroupName <String> -Name <String> -Location <String>
 -HealthCheckProperties <PSHealthCheckStepProperties> [-Tag <Hashtable>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## Beschreibung
Das Cmdlet **New-AzDeploymentManagerStep** erstellt einen Bereitstellungsschritt, auf den in Rollouts verwiesen werden kann.
Geben Sie den *Namen* , die *ResourceGroupName* und die erforderlichen Eigenschaften an.

Sie können das zurückgegebene Objekt lokal ändern und dann die Änderungen auf den Schritt mithilfe des Set-AzDeploymentManagerStep-Cmdlets anwenden.

## Beispiele

### Beispiel 1
```powershell
PS C:\> New-AzDeploymentManagerStep -ResourceGroupName ContosoResourceGroup -Name ContosoService1WaitStep -Location "Central US" -Duration PT20M
```

Erstellt einen Schritt im ContosoResourceGroup mit dem Namen ContosoService1WaitStep mit dem zentralen US-Standort als Speicherort der Ressource. Die Duration-Eigenschaft gibt die Dauer an, die das Rollout wartet, bevor der nächste Schritt ausgeführt wird.

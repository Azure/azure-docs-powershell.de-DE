# Start-AzDataFactoryV2DataFlowDebugSession

## Synopsis
Startet eine Datenfluss-Debugsitzung in Azure Data Factory

## Syntax

### ByFactoryName (Standard)
```
Start-AzDataFactoryV2DataFlowDebugSession [[-IntegrationRuntimeFile] <String>] [-AsJob]
 [-ResourceGroupName] <String> [-DataFactoryName] <String> [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

### ByFactoryObject
```
Start-AzDataFactoryV2DataFlowDebugSession [[-IntegrationRuntimeFile] <String>] [-AsJob]
 [-DataFactory] <PSDataFactory> [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

### ByResourceId
```
Start-AzDataFactoryV2DataFlowDebugSession [[-IntegrationRuntimeFile] <String>] [-AsJob] [-ResourceId] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## Beschreibung
Dieser lange ausgeführte Befehl startet eine Datenfluss-Debug-Sitzung für die bevorstehenden Debug-Befehle. Dieser Befehl kann mit einer Integration Runtime-Definition angefügt werden, um die Größe/den Typ des Debug-Sitzungs Clusters zu konfigurieren.
Die PowerShell-Befehlssequenz für den Workflow zum Debuggen von Datenströmen sollte wie folgt lauten:
1. Start-AzDataFactoryV2DataFlowDebugSession
1. Add-AzDataFactoryV2DataFlowDebugSessionPackage
1. Invoke-AzDataFactoryV2DataFlowDebugSessionCommand (wiederholen Sie diesen Schritt für verschiedene Befehle/Ziele, oder wiederholen Sie Schritt 2-3, um die Paketdatei zu ändern)
1. Stop-AzDataFactoryV2DataFlowDebugSession

# Set-AzSqlDatabaseAuditing

## Synopsis
**Wichtig: Dieses Cmdlet ist veraltet, wenn Sie von [AzSqlDatabaseAudit](https://docs.microsoft.com/en-us/powershell/module/az.sql/set-azsqldatabaseaudit) ersetzt werden.**

## Beschreibung
Das Cmdlet " **Satz-AzSqlDatabaseAuditing** " ändert die Überwachungseinstellungen einer Azure SQL-Datenbank.
Um das Cmdlet zu verwenden, verwenden Sie die Parameter *ResourceGroupName* , *Servername* und *DatabaseName* , um die Datenbank zu identifizieren.
Das Ziel der Überwachungsprotokolle wird bestimmt, indem Sie einen der folgenden Switch-Parameter angeben: BlobStorage, LogAnalytics oder EventHub (wenn keine angegeben ist, ist der Standardwert BlobStorage).
Verwenden Sie den *State* -Parameter, um die Richtlinie zu aktivieren/deaktivieren.
Wenn das Überwachungsprotokoll Ziel BLOB-Speicher ist, geben Sie den *StorageAccountName* -Parameter an, um das Speicherkonto für die Überwachungsprotokolle und den *StorageKeyType* -Parameter zum Definieren der Speicherschlüssel zu ermitteln. Sie können auch die Aufbewahrung für die Überwachungsprotokolle definieren, indem Sie den Wert des *RetentionInDays* -Parameters festlegen, um den Zeitraum für die Überwachungsprotokolle zu definieren.
Wenn das Cmdlet erfolgreich ausgeführt wird und Sie den *passthru* -Parameter verwenden, wird zusätzlich zu den Datenbankbezeichnern ein Objekt zurückgegeben, das die aktuellen Überwachungseinstellungen beschreibt.
Zu den Datenbankbezeichnern gehören **ResourceGroupName** , **Servername** und **DatabaseName** , sind aber nicht darauf limitiert.

## Beispiele

### Beispiel 1: Aktivieren der BLOB-Speicher Überwachungsrichtlinie einer Azure SQL-Datenbank
```
PS C:\>Set-AzSqlDatabaseAuditing -State Enabled -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -StorageAccountName "Storage22" -DatabaseName "Database01"
```

### Beispiel 2: Deaktivieren der BLOB-Speicher Überwachungsrichtlinie einer Azure SQL-Datenbank
```
PS C:\>Set-AzSqlDatabaseAuditing -State Disabled -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -DatabaseName "Database01"
```

### Beispiel 3: Aktivieren der BLOB-Speicher Überwachungsrichtlinie einer Azure SQL-Datenbank unter Verwendung eines speicherkontos aus einem anderen Abonnement
```
PS C:\>Set-AzSqlDatabaseAuditing -State Enabled -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -StorageAccountName "Storage22" -StorageAccountSubscriptionId "7fe3301d-31d3-4668-af5e-211a890ba6e3"
```

### Beispiel 4: Aktivieren der BLOB-Speicher Überwachungsrichtlinie einer Azure SQL-Datenbank mit erweiterter Filterung mithilfe eines T-SQL-Prädikats
```
PS C:\>Set-AzSqlDatabaseAuditing -State Enabled -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -StorageAccountName "Storage22" -DatabaseName "Database01" -PredicateExpression "statement <> 'select 1'"
```

### Beispiel 5: Entfernen der erweiterten Filtereinstellung aus der BLOB-Speicher Überwachungsrichtlinie einer Azure SQL-Datenbank
```
PS C:\>Set-AzSqlDatabaseAuditing -State Enabled -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -StorageAccountName "Storage22" -DatabaseName "Database01" -PredicateExpression ""
```

### Beispiel 6: Aktivieren der Ereignis-Hub-Überwachungsrichtlinie einer Azure SQL-Datenbank
```
PS C:\>Set-AzSqlDatabaseAuditing -State Enabled -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -EventHub -EventHubName "EventHubName" -EventHubAuthorizationRuleResourceId "EventHubAuthorizationRuleResourceId"
```

### Beispiel 7: Deaktivieren der Ereignis-Hub-Überwachungsrichtlinie einer Azure SQL-Datenbank
```
PS C:\>Set-AzSqlDatabaseAuditing -State Disabled -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -EventHub
```

### Beispiel 8: Aktivieren der Überwachungsrichtlinie für die Protokollanalyse einer Azure SQL-Datenbank
```
PS C:\>Set-AzSqlDatabaseAuditing -State Enabled -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -DatabaseName "Database01" -LogAnalytics -WorkspaceResourceId "/subscriptions/4b9e8510-67ab-4e9a-95a9-e2f1e570ea9c/resourceGroups/insights-integration/providers/Microsoft.OperationalInsights/workspaces/viruela2"
```

### Beispiel 9: Deaktivieren der Überwachungsrichtlinie für die Protokollanalyse einer Azure SQL-Datenbank
```
PS C:\>Set-AzSqlDatabaseAuditing -State Disabled -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -DatabaseName "Database01" -LogAnalytics
```

### Beispiel 10: Deaktivieren Sie die Protokollanalyse-Überwachungsrichtlinie einer Azure SQL-Datenbank durch Pipeline.
```
PS C:\>Get-AzSqlDatabase -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -DatabaseName "Database01" | Set-AzSqlDatabaseAuditing -LogAnalytics -State Disabled
```

### -AuditActionGroup
Die empfohlene Gruppe der zu verwendenden Aktionsgruppen ist die folgende Kombination, die alle Abfragen und gespeicherten Prozeduren überprüft, die für die Datenbank ausgeführt werden, sowie erfolgreiche und fehlgeschlagene Anmeldungen:  
  
"BATCH_COMPLETED_GROUP",  
"SUCCESSFUL_DATABASE_AUTHENTICATION_GROUP",  
"FAILED_DATABASE_AUTHENTICATION_GROUP"  
Diese obige Kombination ist auch die standardmäßig konfigurierte Gruppe. Diese Gruppen decken alle SQL-Anweisungen und gespeicherten Prozeduren ab, die für die Datenbank ausgeführt werden, und sollten nicht in Verbindung mit anderen Gruppen verwendet werden, da dies zu doppelten Überwachungsprotokollen führt.
Weitere Informationen finden Sie unter https://docs.microsoft.com/en-us/sql/relational-databases/security/auditing/sql-server-audit-action-groups-and-actions#database-level-audit-action-groups .

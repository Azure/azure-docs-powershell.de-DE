# Enable-AzSqlServerAdvancedDataSecurity

## Synopsis
Ermöglicht erweiterte Datensicherheit auf einem Server.

## Syntax

```
Enable-AzSqlServerAdvancedDataSecurity [-DoNotConfigureVulnerabilityAssessment] [-AsJob]
 [-DeploymentName <String>] [-InputObject <AzureSqlServerModel>] -ServerName <String>
 [-ResourceGroupName] <String> [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## Beschreibung
Das Cmdlet **enable-AzSqlServerAdvancedDataSecurity** ermöglicht erweiterte Datensicherheit auf einem Server. Advanced Data Security ist ein einheitliches Sicherheitspaket, das die Datenklassifizierung, die Gefährdungsbeurteilung und den erweiterten Bedrohungsschutz für Ihren Server umfasst. (Es wird automatisch ein neues Speicherkonto zum Speichern von Sicherheitsrisikobewertungen erstellt. Wenn zuvor ein Speicherkonto für diesen Zweck erstellt wurde, wird es stattdessen verwendet.

## Beispiele

### Beispiel 1: Aktivieren der erweiterten Datensicherheit für Server
```powershell
PS C:\>  Enable-AzSqlServerAdvancedDataSecurity `
            -ResourceGroupName "ResourceGroup01" `
            -ServerName "Server01" 

ResourceGroupName            : ResourceGroup01
ServerName                   : Server01
IsEnabled                    : True
```

### Beispiel 2: Aktivieren der Server erweiterten Datensicherheit von der Server Ressource
```powershell
PS C:\>  Get-AzSqlServer `
           -ResourceGroupName "ResourceGroup01" `
           -ServerName "Server01" `
           | Enable-AzSqlServerAdvancedDataSecurity

ResourceGroupName            : ResourceGroup01
ServerName                   : Server01
IsEnabled                    : True
```

### Beispiel 3

Ermöglicht erweiterte Datensicherheit auf einem Server. automatisch

```powershell
<!-- Aladdin Generated Example --> 
Enable-AzSqlServerAdvancedDataSecurity -DoNotConfigureVulnerabilityAssessment -ResourceGroupName MyResourceGroup -ServerName s1
```

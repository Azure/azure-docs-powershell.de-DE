# Add-AzWebAppTrafficRouting

## Synopsis
Fügen Sie dem Steckplatz eine Routingregel hinzu.

## Syntax

```
Add-AzWebAppTrafficRouting -ResourceGroupName <String> -WebAppName <String> -RoutingRule <Hashtable>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## Beschreibung
Das Cmdlet **Add-AzWebAppTrafficRouting** fügt eine Routing Regel zu einem Azure Web App-Steckplatz hinzu.

## Beispiele

### Beispiel 1: Hinzufügen einer Routingregel zur Übertragung von 15% des Produktions Datenverkehrs an den STG-Steckplatz
```powershell
PS C:\>Add-AzWebAppTrafficRouting -ResourceGroupName "Default-Web-WestUS" -WebAppName "ContosoSite" 
-RoutingRule @{ActionHostName='XXXX.azurewebsites.net';ReroutePercentage=15;Name='Stg'}
```

Dieser Befehl fügt eine Routingregel hinzu, um 15% des Produktions Verkehrs an den STG-Steckplatz zu übertragen.

### Beispiel 2 Fügen Sie eine Routingregel hinzu, um den Produktionsverkehr in den STG-Steckplatz zu übertragen, der sich in inkrementeller Weise von 50% auf 90% erstreckt.
```powershell
PS C:\>Add-AzWebAppTrafficRouting -ResourceGroupName "Default-Web-WestUS" -WebAppName "ContosoSite" 
-RoutingRule @{ActionHostName='XXXX.azurewebsites.net';ReroutePercentage=50;ChangeIntervalInMinutes=1;
MinReroutePercentage=50;MaxReroutePercentage=90;Name='Stg';ChangeStep=10}
```

Mit diesem Befehl wird eine Routingregel hinzugefügt, um den Produktionsverkehr in die STG-Steckplatz Bereiche von 50% auf 90% in inkrementeller Weise zu übertragen.

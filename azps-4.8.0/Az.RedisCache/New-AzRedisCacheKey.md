# New-AzRedisCacheKey

## Synopsis
Regeneriert die Zugriffstaste eines a/a-Caches.

## Syntax

```
New-AzRedisCacheKey [-ResourceGroupName <String>] -Name <String> -KeyType <String> [-Force]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## Beschreibung
Mit dem Cmdlet **New-AzRedisCacheKey** wird die Zugriffstaste eines Azure-a/a-Caches neu generiert.

## Beispiele

### Beispiel 1: Erneutes Generieren eines Primärschlüssels
```
PS C:\>New-AzRedisCacheKey -ResourceGroupName "ResourceGroup03" -Name "myCache" -KeyType "Primary" -Force

          PrimaryKey        : pJ+jruGKPHDKsEC8kmoybobH3TZx2njBR3ipEsquZFo=

          SecondaryKey      : sJ+jruGKPHDKsEC8kmoybobH3TZx2njBR3ipEsquZFo=
```

Mit diesem Befehl wird der Primärschlüssel eines Zeichenfolgen-Caches neu generiert.

### Beispiel 2: Erneutes Generieren eines sekundären Schlüssels
```
PS C:\>New-AzRedisCacheKey -ResourceGroupName "ResourceGroup03" -Name "myCache" -KeyType "Secondary" -Force
          PrimaryKey        : pJ+jruGKPHDKsEC8kmoybobH3TZx2njBR3ipEsquZFo=

          SecondaryKey      : sJ+jruGKPHDKsEC8kmoybobH3TZx2njBR3ipEsquZFo=
```

Mit diesem Befehl wird der sekundäre Schlüssel eines a/a-Caches neu generiert.

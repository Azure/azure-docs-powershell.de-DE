# Update-AzCustomIpPrefix

## Synopsis
Aktualisiert eine CustomIpPrefix

## Syntax

### UpdateByNameParameterSet
```
Update-AzCustomIpPrefix -Name <String> -ResourceGroupName <String> [-Commission] [-Decomission]
 [-Tag <Hashtable>] [-AsJob] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

### UpdateByInputObjectParameterSet
```
Update-AzCustomIpPrefix -InputObject <PSCustomIpPrefix> [-Commission] [-Decomission] [-Tag <Hashtable>]
 [-AsJob] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### UpdateByResourceIdParameterSet
```
Update-AzCustomIpPrefix -ResourceId <String> [-Commission] [-Decomission] [-Tag <Hashtable>] [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## Beschreibung
Das Cmdlet **Update-AzCustomIpPrefix** ermöglicht es dem Benutzer, die CustomIpPrefix zu übernehmen oder die Tags der Ressource zu bearbeiten.

## Beispiele

### Beispiel 1: Commission the CustomIpPrefix
```powershell
PS C:\> Update-AzCustomIpPrefix -Name $prefixName -ResourceGroupName $rgName -Commission
```

Der obige Befehl startet den inbetriebnahmeprozess des CustomIpPrefix.

### Beispiel 2: Außerbetriebnahme des CustomIpPrefix
```powershell
PS C:\> Update-AzCustomIpPrefix -Name $prefixName -ResourceGroupName $rgName -Decommission
```

Mit dem obigen Befehl wird der Deinstallationsvorgang des CustomIpPrefix gestartet.

### Beispiel 3: Aktualisieren von Tags für die CustomIpPrefix
```powershell
PS C:\> Update-AzCustomIpPrefix -Name $prefixName -ResourceGroupName $rgName -Tag $tags
```

Mit dem obigen Befehl werden die Tags für die CustomIpPrefix aktualisiert.

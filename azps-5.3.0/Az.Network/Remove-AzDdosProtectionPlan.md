# Remove-AzDdosProtectionPlan

## SYNOPSIS
Entfernt einen DDoS-Schutzplan.

## SYNTAX

```
Remove-AzDdosProtectionPlan -ResourceGroupName <String> -Name <String> [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESCHREIBUNG
Das Remove-AzDdosProtectionPlan entfernt einen DDoS-Schutzplan.

## BEISPIELE

### Beispiel 1: Entfernen eines leeren DDoS-Schutzplans
```
D:\> Remove-AzDdosProtectionPlan -ResourceGroupName ResourceGroupName -Name DdosProtectionPlan
```

In diesem Fall entfernen wir wie angegeben einen DDoS-Schutzplan.

### Beispiel 2: Entfernen eines DDoS-Schutzplans, der einem virtuellen Netzwerk zugeordnet ist
```
D:\> $vnet = Get-AzVirtualNetwork -Name VnetName -ResourceGroupName ResourceGroupName
D:\> $vnet.DdosProtectionPlan = $null
```

DDoS-Schutzpläne können nicht gelöscht werden, wenn sie einem virtuellen Netzwerk zugeordnet sind. Der erste Schritt besteht also im Entfernen der Zuordnung beider Objekte. Hier erhalten wir die neueste Version des virtuellen Netzwerks, die dem Plan zugeordnet ist, und wir legen die Eigenschaft **"DdosProtectionPlan"** auf einen leeren Wert und das Kennzeichen **"EnableDdosProtection"** (dieses Kennzeichen kann ohne einen Plan nicht wahr sein).
Anschließend speichern wir den neuen Zustand, indem wir die lokale Variable in **Set-AzVirtualNetwork verwenden.** An diesem Punkt ist der Plan nicht mehr mit dem virtuellen Netzwerk verknüpft.
Wenn dies der letzte ist, der dem Plan zugeordnet ist, können Sie den DDoS-Schutzplan mit dem Befehl "Remove-AzDdosProtectionPlan" entfernen.

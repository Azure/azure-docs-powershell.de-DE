# Remove-AzDeploymentManagerServiceTopology

## SYNOPSIS
Löscht die Diensttopologie. Alle dienste, die unter einer Diensttopologie erstellt wurden, müssen gelöscht werden, bevor die Diensttopologie gelöscht wird.

## SYNTAX

### Interaktiv (Standard)
```
Remove-AzDeploymentManagerServiceTopology [-ResourceGroupName] <String> [-Name] <String> [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ResourceId
```
Remove-AzDeploymentManagerServiceTopology [-ResourceId] <String> [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### InputObject
```
Remove-AzDeploymentManagerServiceTopology [-InputObject] <PSServiceTopologyResource> [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESCHREIBUNG
Das **Cmdlet "Remove-AzDeploymentManagerServiceTopology"** löscht eine Diensttopologie.

Geben Sie die Diensttopologie durch ihren Namen und den Namen der Ressourcengruppe an. Alternativ können Sie das Objekt "ServiceTopology" oder "ResourceId" bereitstellen.

## BEISPIELE

### Beispiel 1
```powershell
PS C:\> Remove-AzDeploymentManagerServiceTopology -ResourceGroupName ContosoResourceGroup -Name ContosoServiceTopology
```

Mit diesem Befehl wird eine Diensttopologie namens "ContosoServiceTopology" in der ContosoResourceGroup gelöscht.

### Beispiel 2: Löschen einer Diensttopologie mithilfe des Ressourcenbezeichners.
```powershell
PS C:\> Remove-AzDeploymentManagerServiceTopology -ResourceId "/subscriptions/subscriptionId/resourcegroups/ContosoResourceGroup/providers/Microsoft.DeploymentManager/serviceTopologies/ContosoServiceTopology"
```

Mit diesem Befehl wird eine Diensttopologie mit dem Namen "ContosoServiceTopology" in der ContosoResourceGroup gelöscht.

### Beispiel 3: Löschen einer Diensttopologie mithilfe des Diensttopologieobjekts.
```powershell
PS C:\> Remove-AzDeploymentManagerService -InputObject $serviceTopologyObject
```

Mit diesem Befehl wird eine Diensttopologie gelöscht, deren Name und Ressourcengruppe mit den Eigenschaften "Name" bzw. "ResourceGroupName" des $serviceTopologyObject übereinstimmen.

# Sync-AzDataFactoryV2IntegrationRuntimeCredential

## SYNOPSIS
Synchronisiert Anmeldeinformationen zwischen Integrationslaufzeitknoten.

## SYNTAX

### ByIntegrationRuntimeName (Standard)
```
Sync-AzDataFactoryV2IntegrationRuntimeCredential [-Force] [-IntegrationRuntimeName] <String>
 [-ResourceGroupName] <String> [-DataFactoryName] <String> [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

### ByResourceId
```
Sync-AzDataFactoryV2IntegrationRuntimeCredential [-Force] [-ResourceId] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ByIntegrationRuntimeObject
```
Sync-AzDataFactoryV2IntegrationRuntimeCredential [-Force] [-InputObject] <PSIntegrationRuntime>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESCHREIBUNG
Das **Cmdlet Sync-AzDataFactoryV2IntegrationRuntimeCredential** synchronisiert lokale Anmeldeinformationen zwischen Integrations-Runtime-Knoten, wodurch die Identischkeit der Anmeldeinformationen in allen Knoten erzwingt wird.

## BEISPIELE

### Beispiel 1: Synchronisieren von Integrationslaufzeitanmeldeinformationen

Synchronisiert Anmeldeinformationen zwischen Integrationslaufzeitknoten. (automatisch generiert)

```powershell <!-- Aladdin Generated Example --> 
Sync-AzDataFactoryV2IntegrationRuntimeCredential -DataFactoryName 'test-df-eu2' -IntegrationRuntimeName 'test-selfhost-ir' -ResourceGroupName 'rg-test-dfv2'
```

# Remove-AzSentinelAlertRule

## SYNOPSIS
Löschen sie eine Analyse.

## SYNTAX

### AlertRuleId (Standard)
```
Remove-AzSentinelAlertRule -ResourceGroupName <String> -WorkspaceName <String> -AlertRuleId <String>
 [-PassThru] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### InputObject
```
Remove-AzSentinelAlertRule -InputObject <PSSentinelAlertRule> [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESCHREIBUNG
Das **Cmdlet "Remove-AzSentinelAlertRule"** löscht eine Warnungsregel endgültig aus einem angegebenen Arbeitsbereich.
Sie können ein **"AlertRule"-Objekt** mithilfe des Pipelineoperators übergeben, oder Sie können die erforderlichen Parameter angeben.
Sie können den Parameter "Bestätigen" und $ConfirmPreference Windows PowerShell, um zu steuern, ob sie vom Cmdlet zur Bestätigung aufgefordert werden.

## BEISPIELE

### Beispiel 1
```powershell
PS C:\> Remove-AzSentinelAlertRule -ResourceGroupName "MyResourceGroup" -WorkspaceName "MyWorkspaceName" -AlertRuleId "MyAlertRuleId"
```

Mit diesem Befehl wird die Warnungsregel aus dem Arbeitsbereich entfernt.

# Stop-AzPolicyRemediation

## SYNOPSIS
Bricht eine in Bearbeitung ausgeführte Richtlinienbehebung ab.

## SYNTAX

### ByName (Standard)
```
Stop-AzPolicyRemediation -Name <String> [-Scope <String>] [-ManagementGroupName <String>]
 [-ResourceGroupName <String>] [-AsJob] [-PassThru] [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

### ByResourceId
```
Stop-AzPolicyRemediation -ResourceId <String> [-AsJob] [-PassThru] [-DefaultProfile <IAzureContextContainer>]
 [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ByInputObject
```
Stop-AzPolicyRemediation -InputObject <PSRemediation> [-AsJob] [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESCHREIBUNG
Das **Cmdlet "Stop-AzPolicyRemediation"** bricht eine in Bearbeitung ausgeführte Richtlinienbehebung ab. Aktive Bereitstellungen werden abgebrochen, und es werden keine neuen Bereitstellungen erstellt.

## BEISPIELE

### Beispiel 1: Abbrechen einer Richtlinienbehebung im Bereich "Ressourcengruppe"
```
PS C:\> Stop-AzPolicyRemediation -ResourceGroupName "myRG" -Name "remediation1"
```

Dieser Befehl bricht die Wartung mit dem Namen "remediation1" in der Ressourcengruppe "myRG" ab.

### Beispiel 2: Abbrechen der Wartung einer Verwaltungsgruppe über Piping
```
PS C:\> $remediation = Get-AzPolicyRemediation -ManagementGroupName "mg1" -Name "remediation1"
PS C:\> $remediation | Stop-AzPolicyRemediation
```

Dieser Befehl bricht die Wartung namens "remediation1" in der Verwaltungsgruppe "mg1" ab.

# Remove-AzStorageContainerStoredAccessPolicy

## SYNOPSIS
Entfernt eine gespeicherte Zugriffsrichtlinie aus einem Azure-Speichercontainer.

## SYNTAX

```
Remove-AzStorageContainerStoredAccessPolicy [-Container] <String> [-Policy] <String> [-PassThru]
 [-Context <IStorageContext>] [-ServerTimeoutPerRequest <Int32>] [-ClientTimeoutPerRequest <Int32>]
 [-DefaultProfile <IAzureContextContainer>] [-ConcurrentTaskCount <Int32>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## BESCHREIBUNG
Das **Cmdlet "Remove-AzStorageContainerStoredAccessPolicy"** entfernt eine gespeicherte Zugriffsrichtlinie aus einem Azure-Speichercontainer.

## BEISPIELE

### Beispiel 1: Entfernen einer gespeicherten Zugriffsrichtlinie aus einem Speichercontainer
```
PS C:\>Remove-AzStorageContainerStoredAccessPolicy -Container "MyContainer" -Policy "Policy03"
```

Mit diesem Befehl wird eine Zugriffsrichtlinie mit dem Namen "Policy03" aus dem gespeicherten Container "MyContainer" entfernt.

### -ConcurrentTaskCount
Gibt die maximalen gleichzeitigen Netzwerkanrufe an.
Sie können diesen Parameter verwenden, um die Parallelität zur Einschränkung der lokalen CPU- und Bandbreitennutzung zu beschränken, indem Sie die maximale Anzahl von gleichzeitigen Netzwerkaufrufen angeben.
Der angegebene Wert ist eine absolute Anzahl und wird nicht mit der Anzahl der Kernwerte multipliziert.
Dieser Parameter kann zur Verringerung von Netzwerkverbindungsproblemen in Umgebungen mit geringer Bandbreite beitragen, z. B. 100 Kilobit pro Sekunde.
Der Standardwert ist 10.

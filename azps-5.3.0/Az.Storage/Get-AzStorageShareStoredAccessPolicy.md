# Get-AzStorageShareStoredAccessPolicy

## SYNOPSIS
Ruft gespeicherte Zugriffsrichtlinien für eine Speicherfreigabe ab.

## SYNTAX

```
Get-AzStorageShareStoredAccessPolicy [-ShareName] <String> [[-Policy] <String>] [-Context <IStorageContext>]
 [-ServerTimeoutPerRequest <Int32>] [-ClientTimeoutPerRequest <Int32>]
 [-DefaultProfile <IAzureContextContainer>] [-ConcurrentTaskCount <Int32>] [<CommonParameters>]
```

## BESCHREIBUNG
Das **Cmdlet "Get-AzStorageShareStoredAccessPolicy"** ruft gespeicherte Zugriffsrichtlinien für eine Azure Storage Share ab.
Um eine bestimmte Richtlinie zu erhalten, geben Sie sie nach Name an.

## BEISPIELE

### Beispiel 1: Erhalten einer gespeicherten Zugriffsrichtlinie in einer Freigabe
```
PS C:\>Get-AzStorageShareStoredAccessPolicy -ShareName "ContosoShare" -Policy "GeneralPolicy"
```

Dieser Befehl ruft eine gespeicherte Zugriffsrichtlinie namens "GeneralPolicy" in ContosoShare ab.

### Beispiel 2: Freigeben aller Gespeicherten Zugriffsrichtlinien
```
PS C:\>Get-AzStorageShareStoredAccessPolicy -ShareName "ContosoShare"
```

Dieser Befehl ruft alle Richtlinien für gespeicherten Zugriff in ContosoShare ab.

### -ConcurrentTaskCount
Gibt die maximalen gleichzeitigen Netzwerkanrufe an.
Sie können diesen Parameter verwenden, um die Parallelität zum Drosseln der lokalen CPU- und Bandbreitenverwendung zu beschränken, indem Sie die maximale Anzahl gleichzeitiger Netzwerkaufrufe angeben.
Der angegebene Wert ist eine absolute Anzahl und wird nicht mit der Anzahl der Kernwerte multipliziert.
Dieser Parameter kann zur Verringerung von Netzwerkverbindungsproblemen in Umgebungen mit geringer Bandbreite beitragen, z. B. 100 Kilobit pro Sekunde.
Der Standardwert ist 10.

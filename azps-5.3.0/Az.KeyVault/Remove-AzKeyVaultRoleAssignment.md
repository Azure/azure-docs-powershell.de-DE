# Remove-AzKeyVaultRoleAssignment

## SYNOPSIS
Entfernt eine Rollenzuweisung für den angegebenen Prinzipal, dem eine bestimmte Rolle in einem bestimmten Bereich zugewiesen ist.

## BESCHREIBUNG
Verwenden Sie `Remove-AzKeyVaultRoleAssignment` das Cmdlet, um den Zugriff auf einen Beliebigen Prinzipal in einem bestimmten Bereich und einer bestimmten Rolle zu widerrufen. Das Objekt der Abtretung, d. h. der Prinzipal MUSS angegeben werden. Der Prinzipal kann ein Benutzer (SignInName- oder ObjectId-Parameter verwenden, um einen Benutzer zu identifizieren), eine Sicherheitsgruppe (den ObjectId-Parameter zum Identifizieren einer Gruppe verwenden) oder ein Dienstprinzipal (ApplicationId- oder ObjectId-Parameter verwenden, um einen "ServicePrincipal" zu identifizieren) sein. Die Rolle, der der Prinzipal zugewiesen ist, muss mit dem Parameter "RoleDefinitionName" oder "RoleDefinitionId" angegeben werden.

## BEISPIELE

### Beispiel 1
```powershell
PS C:\> Remove-AzKeyVaultRoleAssignment -HsmName myHsm -RoleDefinitionName "Managed HSM Policy Administrator" -SignInName user1@microsoft.com -Scope "/keys"
```

In diesem Beispiel wird die Rolle "Verwalteter HSM-Richtlinienadministrator" von user1@microsoft.com "" im Bereich "/keys" widerrufen.

### Beispiel 2
```powershell
PS C:\> Get-AzKeyVaultRoleAssignment -HsmName myHsm -SignInName user1@microsoft.com | Remove-AzKeyVaultRoleAssignment
```

In diesem Beispiel werden alle Rollen von " user1@microsoft.com in allen Gültigkeitsbereich widerrufen.

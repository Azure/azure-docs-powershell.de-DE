# Add-AzKeyVaultNetworkRule

## SYNOPSIS
Fügt eine Regel hinzu, die den Zugriff auf einen Schlüsseltresor basierend auf der Internetadresse des Clients einschränken soll.

## SYNTAX

### ByVaultName (Standard)
```
Add-AzKeyVaultNetworkRule [-VaultName] <String> [[-ResourceGroupName] <String>] [-IpAddressRange <String[]>]
 [-VirtualNetworkResourceId <String[]>] [-PassThru] [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

### ByInputObject
```
Add-AzKeyVaultNetworkRule [-InputObject] <PSKeyVault> [-IpAddressRange <String[]>]
 [-VirtualNetworkResourceId <String[]>] [-PassThru] [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

### ByResourceId
```
Add-AzKeyVaultNetworkRule [-ResourceId] <String> [-IpAddressRange <String[]>]
 [-VirtualNetworkResourceId <String[]>] [-PassThru] [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

## BESCHREIBUNG
Das **Cmdlet "Add-AzKeyVaultNetworkRule"** gewährt oder schränkt den Zugriff auf einen Schlüsseltresor auf eine Reihe von Anrufern ein, die von deren IP-Adressen oder dem virtuellen Netzwerk bestimmt werden, zu dem sie gehören. Die Regel kann den Zugriff für andere Benutzer, Anwendungen oder Sicherheitsgruppen einschränken, denen berechtigungen über die Zugriffsrichtlinie erteilt wurden.

Beachten Sie, dass netzwerkregeln nicht über einen beliebigen ip-Bereich `10.0.0.0-10.255.255.255` (private IP-Adressen) hinzugefügt werden können.

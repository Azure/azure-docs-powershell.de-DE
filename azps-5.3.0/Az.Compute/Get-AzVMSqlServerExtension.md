# Get-AzVMSqlServerExtension

## SYNOPSIS
Ruft die Einstellungen für eine SQL Server erweiterung auf einem virtuellen Computer ab.

## SYNTAX

```
Get-AzVMSqlServerExtension [-ResourceGroupName] <String> [-VMName] <String> [[-Name] <String>]
 [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

## BESCHREIBUNG
Das **Cmdlet "Get-AzVMSqlServerExtension"** ruft die Einstellungen der SQL Server-Infrastruktur als Dienst-Agent (IaaS)-Agent auf einem virtuellen Computer ab.

## BEISPIELE

### Beispiel 1: Festlegen der Einstellungen für eine SQL Server auf einem virtuellen Computer
```
PS C:\> Get-AzVMSqlServerExtension -ResourceGroupName "ResourceGroup11" -VMName "ContosoVM07"
ExtensionName        : SqlIaaSAgent
Publisher            : Microsoft.SqlServer.Management
Version              : 1.0
State                : Enable
RoleName             : VMName
AutoPatchingSettings : Microsoft.WindowsAzure.Commands.ServiceManagement.IaaS.Extensions.AutoPatchingSettings
AutoBackupSettings   : Microsoft.WindowsAzure.Commands.ServiceManagement.IaaS.Extensions.AutoBackupSettings
```

Dieser Befehl ruft die Einstellungen der Erweiterung SQL Server auf einem virtuellen Computer namens ContosoVM07 ab.

### Beispiel 2: Erhalten der Einstellungen mithilfe der Pipeline
```
PS C:\> Get-AzVM -ServiceName "Service08" -Name "ContosoVM22" | Get-AzVMSqlServerExtension
ExtensionName        : SqlIaaSAgent
Publisher            : Microsoft.SqlServer.Management
Version              : 1.0
State                : Enable
RoleName             : VMName
AutoPatchingSettings : Microsoft.WindowsAzure.Commands.ServiceManagement.IaaS.Extensions.AutoPatchingSettings
AutoBackupSettings   : Microsoft.WindowsAzure.Commands.ServiceManagement.IaaS.Extensions.AutoBackupSettings
```

Dieser Befehl ruft den virtuellen Computer "ContosoVM22" im Dienst Service08 mithilfe des cmdlets Get-AzVM ab.
Der Befehl übergibt die Ergebnisse mithilfe des Pipelineoperators an das aktuelle Cmdlet.
Der aktuelle Befehl ruft die Einstellungen des SQL Server A0 auf diesem virtuellen Computer ab.

### Beispiel 3: Erhalten der Einstellungen für eine bestimmte SQL Server Version
```
PS C:\> Get-AzVMSqlServerExtension -ResourceGroupName "ResourceGroup11" -VMName "ContosoVM07" -Version "1.0"
ExtensionName        : SqlIaaSAgent
Publisher            : Microsoft.SqlServer.Management
Version              : 1.0
State                : Enable
RoleName             : VMName
AutoPatchingSettings : Microsoft.WindowsAzure.Commands.ServiceManagement.IaaS.Extensions.AutoPatchingSettings
AutoBackupSettings   : Microsoft.WindowsAzure.Commands.ServiceManagement.IaaS.Extensions.AutoBackupSettings
```

Dieser Befehl ruft die Einstellungen der Version 1.0 der SQL Server auf einem virtuellen Computer namens ContosoVM07 ab.

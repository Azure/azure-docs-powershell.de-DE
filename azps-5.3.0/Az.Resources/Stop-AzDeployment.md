# Stop-AzDeployment

## SYNOPSIS
Abbrechen einer laufenden Bereitstellung

## SYNTAX

### StopByDeploymentName (Standard)
```
Stop-AzDeployment [-Name] <String> [-PassThru] [-Pre] [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

### StopByDeploymentId
```
Stop-AzDeployment -Id <String> [-PassThru] [-Pre] [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

### StopByInputObject
```
Stop-AzDeployment -InputObject <PSDeployment> [-PassThru] [-Pre] [-DefaultProfile <IAzureContextContainer>]
 [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESCHREIBUNG
Das **Cmdlet "Stop-AzDeployment"** bricht eine Bereitstellung im Abonnementbereich ab, die gestartet, aber nicht abgeschlossen wurde.
Um eine Bereitstellung zu beenden, muss die Bereitstellung einen unvollständigen Bereitstellungszustand haben, z. B. "Provisioning", und keinen abgeschlossenen Zustand, z. B. "Provisioned" oder "Failed".

Um eine Bereitstellung im Abonnementbereich zu erstellen, verwenden Sie das New-AzDeployment Cmdlet.

Dieses Cmdlet stoppt nur eine ausgeführte Bereitstellung. Verwenden Sie den *Parameter "Name",* um eine bestimmte Bereitstellung zu beenden.

## BEISPIELE

### Beispiel 1
```
PS C:\>Stop-AzDeployment -Name "deployment01"
```

Dieser Befehl bricht eine laufende Bereitstellung "deployment01" im aktuellen Abonnementbereich ab.

### Beispiel 2
```
PS C:\>Get-AzDeployment -Name "deployment01" | Stop-AzDeployment
```

Dieser Befehl ruft die Bereitstellung "deployment01" im aktuellen Abonnementbereich ab und bricht sie ab.

# Start-AzDataMigrationService

## SYNOPSIS
Startet eine Instanz des Azure-Datenbankmigrationsdiensts in einem beendeten Zustand. 

## SYNTAX

### ComponentNameParameterSet (Standard)
```
Start-AzDataMigrationService -ResourceGroupName <String> -Name <String> [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ComponentObjectParameterSet
```
Start-AzDataMigrationService [-InputObject] <PSDataMigrationService> [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ResourceIdParameterSet
```
Start-AzDataMigrationService [-ResourceId] <String> [-PassThru] [-DefaultProfile <IAzureContextContainer>]
 [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESCHREIBUNG
Das Start-AzDataMigrationService-Cmdlet startet eine Instanz des Azure-Datenbankmigrationsdiensts in einem beendeten Zustand. 

## BEISPIELE

### Beispiel 1
```
PS C:\> Start-AzDataMigrationService -ResourceGroupName MyResourceGroup -ServiceName TestService
```

Im vorstehenden Beispiel wird eine Azure Database Migration Service-Instanz mit dem Namen Testdienst in einem beendeten Zustand gestartet, basierend auf dem dienstnamen, der als Eingabe übergeben wurde.

### Beispiel 2
```
PS C:\> Start-AzDataMigrationService -InputObject $TestService
```

Im vorstehenden Beispiel wird eine Azure Database Migration Service-Instanz gestartet, die auf PSDataMigrationService basiert, das als Eingabeparameter übergeben wurde.

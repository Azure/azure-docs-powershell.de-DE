# Set-AzSqlDatabaseSecondary

## SYNOPSIS
Umschaltt eine sekundäre Datenbank als primäre Datenbank, um einen Failover zu starten.

## SYNTAX

### NoOptionsSet (Standard)
```
Set-AzSqlDatabaseSecondary [-DatabaseName] <String> -PartnerResourceGroupName <String> [-AsJob]
 [-ServerName] <String> [-ResourceGroupName] <String> [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

### ByFailoverParams
```
Set-AzSqlDatabaseSecondary [-DatabaseName] <String> -PartnerResourceGroupName <String> [-Failover]
 [-AllowDataLoss] [-AsJob] [-ServerName] <String> [-ResourceGroupName] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESCHREIBUNG
Das **Cmdlet Set-AzSqlDatabaseSecondary** wechselt eine sekundäre Datenbank in die primäre Datenbank, um ein Failover zu initiieren.
Dieses Cmdlet ist als allgemeiner Konfigurationsbefehl konzipiert, ist aber derzeit auf das Initiieren des Failovers beschränkt.
Geben Sie den *Parameter AllowDataLoss* an, um ein Erzwingen eines Failovers während eines Ausfalls zu initiieren.
Sie müssen diesen Parameter nicht angeben, wenn Sie einen geplanten Vorgang ausführen, z. B. einen Wiederherstellungs drill.
Im letzteren Fall wird die sekundäre Datenbank mit dem primären synchronisiert, bevor sie gewechselt wird.

## BEISPIELE

### Beispiel 1: Initiieren eines geplanten Failovers
```powershell
$database = Get-AzSqlDatabase -DatabaseName $databaseName -ResourceGroupName $secondaryResourceGroupName -ServerName $secondaryServerName
$database | Set-AzSqlDatabaseSecondary -PartnerResourceGroupName $primaryResourceGroupName -Failover
```

### Beispiel 2: Initiieren eines erzwungenen Failovers (mit potenziellem Datenverlust)
```powershell
$database = Get-AzSqlDatabase -DatabaseName $databaseName -ResourceGroupName $secondaryResourceGroupName -ServerName $secondaryServerName
$database | Set-AzSqlDatabaseSecondary -PartnerResourceGroupName $primaryResourceGroupName -Failover -AllowDataLoss
```

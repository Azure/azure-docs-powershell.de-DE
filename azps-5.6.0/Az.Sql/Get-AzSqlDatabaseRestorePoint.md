# Get-AzSqlDatabaseRestorePoint

## SYNOPSIS
Ruft die eindeutigen Wiederherstellungspunkte ab, aus denen ein SQL Data Warehouse wiederhergestellt werden kann.

## SYNTAX

```
Get-AzSqlDatabaseRestorePoint [-ServerName] <String> [-DatabaseName] <String> [-ResourceGroupName] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESCHREIBUNG
Das **Get-AzSqlDatabaseRestorePoint-Cmdlet** ruft die eindeutigen Wiederherstellungspunkte ab, aus der ein Azure SQL Data Warehouse wiederhergestellt werden kann.
Für eine Azure SQL-Datenbank ist das Wiederherstellungsfenster fortlaufend.
Dies bedeutet, dass ein beliebiger Zeitpunkt im Aufbewahrungszeitraum der Sicherung der Datenbank als Wiederherstellungspunkt verwendet werden kann.
Dieses Cmdlet wird auch vom SQL Server Stretch Database Service in Azure unterstützt.

## BEISPIELE

### Beispiel 1: Alle Wiederherstellungspunkte erhalten
```
PS C:\>Get-AzSqlDatabaseRestorePoint -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -DatabaseName "Database01"
ResourceGroupName        : resourcegroup01
ServerName               : server01
DatabaseName             : database01
Location                 : Central US
RestorePointType         : CONTINUOUS
RestorePointCreationDate : 
EarliestRestoreDate      : 8/12/2015 12:00:00 AM
RestorePointLabel        : RestorePoint01
```

Dieser Befehl gibt alle verfügbaren Wiederherstellungspunkte für die Azure SQL Datenbank mit dem Namen Datenbank01 zurück.

# Set-AzSqlServerTransparentDataEncryptionProtector

## SYNOPSIS
Legt den Schutz der transparenten Datenverschlüsselung (Transparent Data Encryption, TDE) für einen SQL fest.

## SYNTAX

```
Set-AzSqlServerTransparentDataEncryptionProtector [-Type] <EncryptionProtectorType> [[-KeyId] <String>]
 [-Force] [-AsJob] [-ServerName] <String> [-ResourceGroupName] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESCHREIBUNG
Das Set-AzSqlServerTransparentDataEncryptionProtector-Cmdlet legt den TDE-Protektor für einen SQL fest.
Durch Ändern des TdE-Schutztyps wird der Schutzschutz gedreht.

## BEISPIELE

### Beispiel 1: Festlegen des Schutztyps für die transparente Datenverschlüsselung (Transparent Data Encryption, TDE) auf ServiceManaged
```
PS C:\> Set-AzSqlServerTransparentDataEncryptionProtector -Type ServiceManaged -ServerName 'ContosoServer' -ResourceGroup 'ContosoResourceGroup'
```

Mit diesem Befehl wird der TdE-Schutztyp eines Servers auf Dienst verwaltet aktualisiert.
ResourceGroupName ServerName Type ServerKeyVaultKeyName
----------------- ----------                   ---- ---------------------
ContosoResourceGroup ContosoServer ServiceManaged ServiceManaged

### Beispiel 2: Festlegen des Schutztyps "Transparente Datenverschlüsselung" auf Azure Key Vault
```
PS C:\> Set-AzSqlServerTransparentDataEncryptionProtector -Type AzureKeyVault -KeyId 'https://contoso.vault.azure.net/keys/contosokey/01234567890123456789012345678901' -ServerName 'ContosoServer' -ResourceGroup 'ContosoResourceGroup'
```

Mit diesem Befehl wird ein Server aktualisiert, um den Serverschlüsseltresorschlüssel mit id ' https://contoso.vault.azure.net/keys/contosokey/01234567890123456789012345678901 ' als TDE-Schutz zu verwenden.
ResourceGroupName ServerName Type ServerKeyVaultKeyName
----------------- ----------                   ---- ---------------------
ContosoResourceGroup ContosoServer AzureKeyVault contoso_contosokey_01234567890123456789012345678901

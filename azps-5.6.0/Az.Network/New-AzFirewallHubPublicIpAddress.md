# New-AzFirewallHubPublicIpAddress

## SYNOPSIS
Öffentliche Ip-Zuordnung zur Firewall auf virtuellem Hub

## SYNTAX

```
New-AzFirewallHubPublicIpAddress [-Count <Int32>] [-Addresses <PSAzureFirewallPublicIpAddress[]>]
 [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

## BESCHREIBUNG
Öffentliche Ip-Zuordnung zur Firewall auf virtuellem Hub

## BEISPIELE

### Beispiel 1
```powershell
PS C:\> New-AzFirewallHubPublicIpAddress -Count 2
```

Dadurch werden zwei öffentliche IPS für die firewall erstellt, die an den virtuellen Hub angefügt sind. Dadurch wird die IP-Adresse im Back-End erstellt. Wir können die ipaddresses nicht explizit für eine neue Firewall bereitstellen.

### Beispiel 2
```powershell
PS C:\> $publicIp1 = New-AzFirewallPublicIpAddress -Address 10.2.3.4
PS C:\> $publicIp2 = New-AzFirewallPublicIpAddress -Address 20.56.37.46
PS C:\> New-AzFirewallHubPublicIpAddress -Count 3 -Addresses $publicIp1, $publicIp2
```

Dadurch wird eine neue öffentliche IP für die Firewall erstellt, indem $publicIp 1, $publicIp 2 beibehalten wird, die bereits in der Firewall vorhanden sind.

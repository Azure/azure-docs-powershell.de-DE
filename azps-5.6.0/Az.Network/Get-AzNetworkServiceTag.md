# Get-AzNetworkServiceTag

## SYNOPSIS
Ruft die Liste der Diensttaginformationsressourcen ab.

## SYNTAX

```
Get-AzNetworkServiceTag -Location <String> [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

## BESCHREIBUNG
Das **Get-AzNetworkServiceTag-Cmdlet** ruft die Liste der Diensttag-Informationsressourcen ab.

Beachten Sie, dass die von Ihnen angegebenen Informationen zur Azure-Region als Referenz für die Version (nicht als Filter basierend auf dem Standort) verwendet werden. Selbst wenn Sie beispielsweise angeben, erhalten Sie die Liste der Diensttags mit Präfixdetails für alle Regionen, jedoch beschränkt auf die Cloud, zu der Ihr Abonnement gehört `-Location eastus2` (d. h. öffentlich, US-Regierung, China oder Deutschland).

### Beispiel 2: Alle Adresspräfixe für AzureSQL erhalten

Der erste Befehl ruft die Liste der Diensttaginformationsressourcen ab und speichert sie in variabler `serviceTags` .
Der zweite Befehl filtert die Liste, um die Informationsressource für Sql auszuwählen.

Der erste Befehl ruft die Liste der Diensttaginformationsressourcen ab und speichert sie in variabler `serviceTags` .

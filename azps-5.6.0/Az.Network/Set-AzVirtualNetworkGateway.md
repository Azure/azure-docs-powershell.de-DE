## BESCHREIBUNG
Das **Cmdlet Set-AzVirtualNetworkGateway** aktualisiert ein Gateway für virtuelle Netzwerke.

## BEISPIELE

### Beispiel 1: Aktualisieren des ASN eines virtuellen Netzwerkgateways
```
PS C:\>$Gateway = Get-AzVirtualNetworkGateway -ResourceGroupName "ResourceGroup001" -Name "Gateway001"
PS C:\> Set-AzVirtualNetworkGateway -VirtualNetworkGateway $Gateway -Asn 1337
```

Der erste Befehl ruft ein virtuelles Netzwerkgateway namens Gateway01 ab, das zur Ressourcengruppe ResourceGroup001 gehört, und speichert es in der Variablen $Gateway Mit dem zweiten Befehl wird das virtuelle Netzwerkgateway aktualisiert, das in variablen $Gateway.
Der Befehl legt außerdem den ASN auf 1337 fest.

### Beispiel 2: Hinzufügen einer IPsec-Richtlinie zu einem Gateway für virtuelle Netzwerke
```
PS C:\>$Gateway = Get-AzVirtualNetworkGateway -ResourceGroupName "ResourceGroup001" -Name "Gateway001"
PS C:\> $vpnclientipsecpolicy = New-AzVpnClientIpsecPolicy -IpsecEncryption AES256 -IpsecIntegrity SHA256 -SALifeTimeSeconds 86472 -SADataSizeKilobytes 429497 -IkeEncryption AES256 -IkeIntegrity SHA256 -DhGroup DHGroup2 -PfsGroup None
PS C:\> $gateway = Set-AzVirtualNetworkGateway -VirtualNetworkGateway $Gateway -VpnClientIpsecPolicy $vpnclientipsecpolicy
```

Der erste Befehl ruft ein virtuelles Netzwerkgateway namens Gateway01 ab, das zur Ressourcengruppe ResourceGroup001 gehört, und speichert es in der Variablen mit dem Namen $Gateway Der zweite Befehl erstellt das Vpn-ipsec-Richtlinienobjekt nach den angegebenen ipsec-Parametern.
Mit dem dritten Befehl wird das gateway für virtuelle Netzwerke aktualisiert, das in variablen $Gateway.
Der Befehl legt außerdem die benutzerdefinierte VPN-IPSec-Richtlinie fest, die im $vpnclientipsecpolicy -Objekt auf dem Gateway für virtuelle Netzwerke angegeben ist.

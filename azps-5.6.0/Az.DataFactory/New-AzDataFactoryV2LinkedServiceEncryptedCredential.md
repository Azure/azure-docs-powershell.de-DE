# New-AzDataFactoryV2LinkedServiceEncryptedCredential

## SYNOPSIS
Verschlüsseln Sie Anmeldeinformationen im verknüpften Dienst mit einer angegebenen Integrationslaufzeit.

## SYNTAX

### ByFactoryName (Standard)
```
New-AzDataFactoryV2LinkedServiceEncryptedCredential [-IntegrationRuntimeName] <String>
 [-DefinitionFile] <String> [-Force] [-ResourceGroupName] <String> [-DataFactoryName] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ByFactoryObject
```
New-AzDataFactoryV2LinkedServiceEncryptedCredential [-IntegrationRuntimeName] <String>
 [-DefinitionFile] <String> [-Force] [-DataFactory] <PSDataFactory> [-DefaultProfile <IAzureContextContainer>]
 [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESCHREIBUNG
Das New-AzDataFactoryV2LinkedServiceEncryptedCredential-Cmdlet verschlüsselt Anmeldeinformationen im verknüpften Dienst mit angegebener Integrationslaufzeit.

Stellen Sie sicher, dass die folgenden Voraussetzungen erfüllt sind:
* **Die Option für** den Remotezugriff ist für die selbst gehostete Integrationslaufzeit aktiviert.
* Powershell 7.0 oder höher wird zum Ausführen des Cmdlets verwendet.

## BEISPIELE

### Beispiel 1: Verschlüsseln von Anmeldeinformationen in einem verknüpften Dienst

Mit diesem Befehl werden die Anmeldeinformationen in C:\samples\WikiSample\TaxiDemo1.jsmit der Integrations-Runtime mit dem Namen test-selfhost-ir verschlüsselt.

```powershell <!-- Aladdin Generated Example --> 
New-AzDataFactoryV2LinkedServiceEncryptedCredential -DataFactoryName WikiADF -DefinitionFile 'C:\samples\WikiSample\TaxiDemo1.json' -IntegrationRuntimeName 'test-selfhost-ir' -ResourceGroupName MyResourceGroup
```

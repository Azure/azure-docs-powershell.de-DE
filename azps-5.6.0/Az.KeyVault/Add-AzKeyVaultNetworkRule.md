# Add-AzKeyVaultNetworkRule

## SYNOPSIS
Fügt eine Regel hinzu, die den Zugriff auf einen Schlüsseltresor basierend auf der Internetadresse des Clients einschränken soll.

## SYNTAX

### ByVaultName (Standard)
```
Add-AzKeyVaultNetworkRule [-VaultName] <String> [[-ResourceGroupName] <String>] [-IpAddressRange <String[]>]
 [-VirtualNetworkResourceId <String[]>] [-PassThru] [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

### ByInputObject
```
Add-AzKeyVaultNetworkRule [-InputObject] <PSKeyVault> [-IpAddressRange <String[]>]
 [-VirtualNetworkResourceId <String[]>] [-PassThru] [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

### ByResourceId
```
Add-AzKeyVaultNetworkRule [-ResourceId] <String> [-IpAddressRange <String[]>]
 [-VirtualNetworkResourceId <String[]>] [-PassThru] [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

## BESCHREIBUNG
Das **Add-AzKeyVaultNetworkRule-Cmdlet** gewährt oder schränkt den Zugriff auf einen Schlüsseltresor auf eine Gruppe von Anrufern ein, die von ihren IP-Adressen oder dem virtuellen Netzwerk bestimmt sind, dem sie angehören. Die Regel kann den Zugriff für andere Benutzer, Anwendungen oder Sicherheitsgruppen einschränken, denen über die Zugriffsrichtlinie Berechtigungen erteilt wurden.

Beachten Sie, dass ein beliebiger IP-Bereich `10.0.0.0-10.255.255.255` innerhalb (private IP-Adressen) nicht zum Hinzufügen von Netzwerkregeln verwendet werden kann.

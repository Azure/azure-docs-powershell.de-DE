# Get-AzInsightsPrivateLinkScopedResource

## SYNOPSIS
Für ressource mit privatem Linkbereich erhalten

## SYNTAX

### ByScopeParameterSet (Standard)
```
Get-AzInsightsPrivateLinkScopedResource -ResourceGroupName <String> -ScopeName <String> [-Name <String>]
 [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

### ByInputObjectParameterSet
```
Get-AzInsightsPrivateLinkScopedResource [-Name <String>] -InputObject <PSMonitorPrivateLinkScope>
 [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

### ByResourceIdParameterSet
```
Get-AzInsightsPrivateLinkScopedResource -ResourceId <String> [-DefaultProfile <IAzureContextContainer>]
 [<CommonParameters>]
```

## BESCHREIBUNG
Eine Liste oder Eine Liste für die Ressource mit privatem Linkbereich kann der Arbeitsbereich "Log Analytics" oder "Application Insights"-Komponente sein.

## BEISPIELE

### Beispiel 1
```powershell
Get-AzInsightsPrivateLinkScopedResource -ResourceGroupName "rg_name" -ScopeName "scope_name"
```

Auflisten einer Ressource mit Bereichsbereich unter privatem Linkbereich "scope_name"

### Beispiel 2
```powershell
Get-AzInsightsPrivateLinkScopedResource -ResourceGroupName "rg_name" -ScopeName "scope_name" -Name "scoped_resource_name"
```

Ressource mit Bereichsbereich unter privatem Linkbereich "scope_name" mit dem Namen "scoped_resource_name"

# New-AzSupportTicket

## SYNOPSIS
Erstellt ein Supportticket.

 -CustomerCountry <String> -CustomerPreferredSupportLanguage <String> [-ProblemStartTime <DateTime>]

## BESCHREIBUNG
Dieses Cmdlet kann zum Erstellen eines Supporttickets für Abrechnungs-, Abonnementverwaltungs-, Kontingent- oder technische Probleme verwendet werden. Verwenden Get-AzSupportService und Get-AzSupportProblemClassification, um den Azure-Dienst und die entsprechenden Problemklassifizierungen zu identifizieren, für die Sie Support anfordern möchten. Sie müssen die folgenden Parameter angeben: 

    • Title
    • Description
    • Severity level
    • ProblemClassificationId
    • CustomerContactDetail (or individual customer contact parameters)

Sie können das New-AzSupportContactProfileObject verwenden, um das CustomerContactDetail-Objekt zu erstellen.

Cloudlösungsanbieter können ein Supportticket für die Abonnements ihres Kunden erstellen, indem sie sich beim Mandanten des Kunden anmelden und mithilfe des *Parameters CSPHomeTenantId* ihre #A0 angeben.

__Für technische Tickets:__

Um den Ressourcennamen anzugeben, geben Sie die ARM-Ressourcen-ID der Ressource mithilfe des *Parameters TechnicalTicketResourceId* an. Sehen Sie sich ein Beispiel unten an. 

__Für Kontingenttickets:__

Wenn Sie eine Kontingenterhöhung für **Compute VM Cores**, **Batch**, **SQL Database** und SQL **Data Warehouse** anfordern möchten, geben Sie weitere Details unter *QuotaTicketDetail-Objekt* an. Das QuotaTicketDetail-Objekt besteht aus drei Eigenschaften, wie unten beschrieben. Eine ausführliche Dokumentation finden Sie [hier.](https://aka.ms/supportrpquotarequestpayload)

    • QuotaChangeRequestSubType

        This is required for certain quota types when there is a sub type that you are requesting quota increase for. Example: Batch, SQL Database and SQL Data Warehouse have a sub type.

    • QuotaChangeRequestVersion

        This is required and indicates the version of the quota change request payload.

    • QuotaChangeRequests

        This is required and is a list of PSQuotaChangeRequest objects. PSQuotaChangeRequest object has 2 required properties.

        ○ Region

            This is the Azure location or region for which you are requesting quota increase. This is the Location property of Get-AzLocation cmdlet.
        
        ○ Payload

            This is where you specify the new limits for the selected quota type.


Ausführliche Dokumentation zum Erstellen von Nutzlast für verschiedene Kontingenttypen finden Sie [hier.](https://aka.ms/supportrpquotarequestpayload)

Sie müssen dann den -CSPHomeTenantId-Parameter verwenden, um ihre #A0 zum Zeitpunkt der Erstellung eines Supporttickets anzugeben.

PS C:\> Login-AzAccount

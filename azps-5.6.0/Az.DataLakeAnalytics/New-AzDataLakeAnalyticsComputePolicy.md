# New-AzDataLakeAnalyticsComputePolicy

## SYNOPSIS
Erstellt eine Data Lake Analytics-Computerichtlinienregel für eine bestimmte AAD-Entität.

## SYNTAX

```
New-AzDataLakeAnalyticsComputePolicy [-ResourceGroupName <String>] [-Account] <String> [-Name] <String>
 [-ObjectId] <Guid> [-ObjectType] <String> [-MaxAnalyticsUnitsPerJob <Int32>] [-MinPriorityPerJob <Int32>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESCHREIBUNG
Die **New-AzDataLakeAnalyticsComputePolicy** erstellt die angegebene Computerichtlinienregel für eine bestimmte AAD-Entität in einem Azure Data Lake Analytics-Konto.

## BEISPIELE

### Beispiel 1: Erstellen einer Computerichtlinie mit nur einer Regel
```
PS C:\>New-AzDataLakeAnalyticsComputePolicy -Account "contosoadla" -Name "myPolicy" -ObjectId 83cb7ad2-3523-4b82-b909-d478b0d8aea3 -ObjectType User -MaxAnalyticsUnitsPerJob 5
```

Mit diesem Befehl wird eine Richtlinie mit dem Namen "myPolicy" im Konto "contosoadla" für den Benutzer mit der ID "83cb7ad2-3523-4b82-b909-d478b0d8aea3" erstellt, die sicherstellt, dass er keinen Auftrag mit mehr als 5 Analyseeinheiten übermitteln kann.

### Beispiel 2: Erstellen einer Computerichtlinie mit beiden Festgelegten Regeln
```
PS C:\>New-AzDataLakeAnalyticsComputePolicy -Account "contosoadla" -Name "myPolicy" -ObjectId 83cb7ad2-3523-4b82-b909-d478b0d8aea3 -ObjectType User -MaxAnalyticsUnitsPerJob 5 -MinPriorityPerJob 100
```

Mit diesem Befehl wird eine Richtlinie mit dem Namen "myPolicy" im Konto "contosoadla" für den Benutzer mit der ID "83cb7ad2-3523-4b82-b909-d478b0d8aea3" erstellt, die sicherstellt, dass keine Aufgaben mit mehr als 5 Analyseeinheiten oder mit einer Priorität unter 100 gesendet werden können.

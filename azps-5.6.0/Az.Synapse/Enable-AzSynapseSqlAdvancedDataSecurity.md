# Enable-AzSynapseSqlAdvancedDataSecurity

## SYNOPSIS
Aktiviert Advanced Data Security in einem Arbeitsbereich.

## SYNTAX

### EnableByNameParameterSet (Standard)
```
Enable-AzSynapseSqlAdvancedDataSecurity [-ResourceGroupName <String>] -WorkspaceName <String>
 [-DoNotConfigureVulnerabilityAssessment] [-DeploymentName <String>] [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### EnableByInputObjectParameterSet
```
Enable-AzSynapseSqlAdvancedDataSecurity -InputObject <PSSynapseWorkspace>
 [-DoNotConfigureVulnerabilityAssessment] [-DeploymentName <String>] [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### EnableByResourceIdParameterSet
```
Enable-AzSynapseSqlAdvancedDataSecurity -ResourceId <String> [-DoNotConfigureVulnerabilityAssessment]
 [-DeploymentName <String>] [-AsJob] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## BESCHREIBUNG
Das **Cmdlet Enable-AzSynapseSqlAdvancedDataSecurity** ermöglicht Advanced Data Security in einem Arbeitsbereich. Advanced Data Security ist ein einheitliches Sicherheitspaket, das Datenklassifizierung, Vulnerability Assessment und Advanced Threat Protection für Ihren Arbeitsbereich umfasst. (Zum Speichern von Sicherheitsbewertungen wird automatisch ein neues Speicherkonto erstellt. Wenn zuvor ein Speicherkonto zu diesem Zweck erstellt wurde, wird es stattdessen verwendet.

## BEISPIELE

### Beispiel 1
```powershell
PS C:\> Enable-AzSynapseSqlAdvancedDataSecurity -WorkspaceName ContosoWorkspace
```

Mit diesem Befehl wird die erweiterte Datensicherheit des Arbeitsbereichs aktiviert.

### Beispiel 2
```powershell
PS C:\> Get-AzSynapseWorkspace -Name ContosoWorkspace | Enable-AzSynapseSqlAdvancedDataSecurity
```

Mit diesem Befehl wird die erweiterte Datensicherheit des Arbeitsbereichs über die Pipeline aktiviert.

### Beispiel 3
```powershell
PS C:\> Enable-AzSynapseSqlAdvancedDataSecurity -WorkspaceName ContosoWorkspace -DoNotConfigureVulnerabilityAssessment
```

Mit diesem Befehl wird die erweiterte Datensicherheit des Arbeitsbereichs über die Pipeline aktiviert, und die Sicherheitsbeurteilung wird nicht automatisch aktiviert.

# Restart-AzVmss

## SYNOPSIS
Startet den VMSS oder einen virtuellen Computer innerhalb des VMSS neu.

## SYNTAX

```
Restart-AzVmss [-ResourceGroupName] <String> [-VMScaleSetName] <String> [[-InstanceId] <String[]>] [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESCHREIBUNG
Das **Cmdlet Restart-AzVmss** startet den VmSS (Virtual Machine Scale Set) neu.
Dieses Cmdlet kann auch verwendet werden, um einen bestimmten virtuellen Computer innerhalb des VMSS mit dem *Parameter InstanzId neu zu* starten.

## BEISPIELE

### Beispiel 1: Neustarten des VMSS
```
PS C:\> Restart-AzVmss -ResourceGroupName "Group001" -VMScaleSetName "VMSS001";
```

Mit diesem Befehl wird der VMSS mit dem Namen VMSS001 neu gestartet, der zur Ressourcengruppe "Group001" gehört.

### Beispiel 2: Neustart eines bestimmten virtuellen Computers im VMSS
```
PS C:\> Restart-AzVmss -ResourceGroupName "Group004" -VMScaleSetName "VMSS001" -InstanceId "1"
```

Mit diesem Befehl wird ein virtueller Computer neu gestartet, der die Instanz-ID 1 im VMSS mit dem Namen VMSS001 hat, der zur Ressourcengruppe "Group001" gehört.

# Set-AzVM

## SYNOPSIS
Kennzeichnet einen virtuellen Computer als generalisiert.

## SYNTAX

### GeneralizeResourceGroupNameParameterSetName (Standard)
```
Set-AzVM [-ResourceGroupName] <String> [-Name] <String> [-Generalized] [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

### RedeployResourceGroupNameParameterSetName
```

 [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

### ReapplyResourceGroupNameParameterSetName

 [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

```

```

### RedeployIdParameterSetName
```
Set-AzVM [-Id] <String> [-Redeploy] [-AsJob] [-NoWait] [-DefaultProfile <IAzureContextContainer>]
 [<CommonParameters>]
```

### ReapplyIdParameterSetName

### SimulateEvictionIdParameterSetName

 [<CommonParameters>]
```

## BESCHREIBUNG
Das **Set-AzVM-Cmdlet** kennzeichnet einen virtuellen Computer als generalisiert.
Melden Sie sich vor dem Ausführen dieses Cmdlets beim virtuellen Computer an, und verwenden Sie Sysprep, um die Festplatte vorzubereiten.

## BEISPIELE

### Beispiel 1: Kennzeichnen eines virtuellen Computers als generalisiert
```
PS C:\> Set-AzVM -ResourceGroupName "ResourceGroup11" -Name "VirtualMachine07" -Generalized
```

Dieser Befehl kennzeichnet den virtuellen Computer mit dem Namen VirtualMachine07 als generalisiert.

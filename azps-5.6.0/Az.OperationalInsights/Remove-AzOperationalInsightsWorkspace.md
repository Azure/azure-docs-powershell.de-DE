# Remove-AzOperationalInsightsWorkspace

## SYNOPSIS
Entfernt einen Arbeitsbereich.

## SYNTAX

```
Remove-AzOperationalInsightsWorkspace [-ResourceGroupName] <String> [-Name] <String> [-ForceDelete] [-Force]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESCHREIBUNG
Das **Cmdlet Remove-AzOperationalInsightsWorkspace** löscht einen vorhandenen Arbeitsbereich.
Wenn dieser Arbeitsbereich zum Zeitpunkt der Erstellung über den *Parameter CustomerId* mit einem vorhandenen Konto verknüpft wurde, wird das ursprüngliche Konto im Portal Operational Insights nicht gelöscht.

## BEISPIELE

### Beispiel 1: Entfernen eines Arbeitsbereichs nach Name
```
PS C:\>Remove-AzOperationalInsightsWorkspace -ResourceGroupName "ContosResourceGroup" -Name "MyWorkspace"
```

Mit diesem Befehl wird der Arbeitsbereich "MyWorkspace" aus der Ressourcengruppe "ContosoResourceGroup" entfernt.

### Beispiel 2: Entfernen eines Arbeitsbereichs mithilfe der Pipeline und ohne Bestätigung
```
PS C:\>Get-AzOperationalInsightsWorkspace -ResourceGroupName "ContosResourceGroup" -Name "MyWorkspace" | Remove-AzOperationalInsightsWorkspace -Force
```

Dieser Befehl verwendet das Get-AzOperationalInsightsWorkspace-Cmdlet, um den Arbeitsbereich mit dem Namen MyWorkspace zu erhalten, und übergibt ihn dann mithilfe des Pipelineoperators an das **Remove-AzOperationalInsightsWorkspace-Cmdlet,** um ihn zu entfernen.
Da der *Parameter Erzwingen* angegeben ist, werden Sie vom Befehl vor dem Entfernen des Arbeitsbereichs nicht aufgefordert.

### Beispiel 3: Erzwingen des Löscharbeitsbereichs (kann nicht wiederhergestellt werden)
```
PS C:\> $workspace = New-AzOperationalInsightsWorkspace -ResourceGroupName $rgname -Name $wsname -Location $wslocation
PS C:\> $workspace | Remove-AzOperationalInsightsWorkspace -ForceDelete
```

Erzwingen sie das Löschen eines Arbeitsbereichs.

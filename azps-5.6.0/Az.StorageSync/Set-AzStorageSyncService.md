# Set-AzStorageSyncService

## SYNOPSIS
Mit diesem Befehl wird der Speichersynchronisierungsdienst in einer Ressourcengruppe definiert.

## SYNTAX

```
Set-AzStorageSyncService [-ResourceGroupName] <String> [-Name] <String> [-Location] <String> [-Tag <Hashtable>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESCHREIBUNG
Ein Speichersynchronisierungsdienst ist die Ressource auf oberster Ebene für Azure File Sync. Mit diesem Befehl wird der Speichersynchronisierungsdienst in einer Ressourcengruppe definiert. Es wird empfohlen, so wenige Speichersynchronisierungsdienste wie unbedingt erforderlich zu erstellen, um verschiedene Servergruppen in Ihrer Organisation voneinander zu unterscheiden. Ein Speichersynchronisierungsdienst enthält Synchronisierungsgruppen und arbeitet auch als Ziel, um Ihre Server zu registrieren. Ein Server kann nur bei einem einzelnen Speichersynchronisierungsdienst registriert werden. Wenn Server jemals an der Synchronisierung derselben Gruppe von Dateien teilnehmen müssen, registrieren Sie sie beim gleichen Speichersynchronisierungsdienst.

## BEISPIELE

### Beispiel 1
```powershell
PS C:\> Set-AzStorageSyncService -ResourceGroupName "myResourceGroup" -StorageSyncServiceName "myStorageSyncServiceName" -IncomingTrafficPolicy "AllowAllTraffic"
```

Mit diesem Befehl wird ein Speichersynchronisierungsdienst festgelegt.

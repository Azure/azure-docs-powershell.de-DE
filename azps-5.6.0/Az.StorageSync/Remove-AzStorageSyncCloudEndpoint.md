# Remove-AzStorageSyncCloudEndpoint

## SYNOPSIS
Mit diesem Befehl wird der angegebene Cloudendpunkt aus einer Synchronisierungsgruppe gelöscht. Ohne mindestens einen Cloudendpunkt können keine anderen Serverendpunkte in dieser Synchronisierungsgruppe synchronisiert werden.

## SYNTAX

### StringParameterSet (Standard)
```
Remove-AzStorageSyncCloudEndpoint [-ResourceGroupName] <String> [-StorageSyncServiceName] <String>
 [-SyncGroupName] <String> [-Name] <String> [-Force] [-PassThru] [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### InputObjectParameterSet
```
Remove-AzStorageSyncCloudEndpoint [-InputObject] <PSCloudEndpoint> [-Force] [-PassThru] [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ResourceIdParameterSet
```
Remove-AzStorageSyncCloudEndpoint [-ResourceId] <String> [-Force] [-PassThru] [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESCHREIBUNG
Mit diesem Befehl wird der angegebene Cloudendpunkt aus einer Synchronisierungsgruppe gelöscht. Die Azure-Datei teilt die Cloudendpunktverweise bleibt von diesem Prozess unberührt. Dieser Befehl ist nur für die Außerbetriebnahme vorgesehen. Das Entfernen eines Cloudendpunkts ist ein zerstörerischer Vorgang. Serverendpunkte können nicht synchronisiert werden, ohne dass mindestens ein Cloudendpunkt vorhanden ist. Dieser Vorgang sollte nicht ausgeführt werden, um Synchronisierungsprobleme zu beheben. Wenn diese Azure-Dateifreigabe erneut zur gleichen Synchronisierungsgruppe wie ein Cloudendpunkt hinzugefügt wird, kann dies zu Konfliktdateien und anderen unbeabsichtigten Folgen führen.

## BEISPIELE

### Beispiel 1
```powershell
PS C:\> Remove-AzStorageSyncCloudEndpoint -Force -ResourceGroupName "myResourceGroup" -StorageSyncServiceName "myStorageSyncServiceName" -SyncGroupName "mySyncGroupName" -Name "myCloudEndpointName"
```

Mit diesem Befehl wird der Cloudendpunkt entfernt.

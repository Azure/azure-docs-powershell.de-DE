# Invoke-AzStorageSyncFileRecall

## SYNOPSIS
Dieser Befehl ruft alle mehrstufigen Dateien wieder auf den lokalen Datenträger zurück.

## SYNTAX

### StringParameterSet (Standard)
```
Invoke-AzStorageSyncFileRecall [-ResourceGroupName] <String> [-StorageSyncServiceName] <String>
 [-SyncGroupName] <String> [-Name] <String> [-Pattern <String>] [-RecallPath <String>] [-PassThru] [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ResourceIdParameterSet
```
Invoke-AzStorageSyncFileRecall [-ResourceId] <String> [-Pattern <String>] [-RecallPath <String>] [-PassThru]
 [-AsJob] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ObjectParameterSet
```
Invoke-AzStorageSyncFileRecall [-InputObject] <PSServerEndpoint> [-Pattern <String>] [-RecallPath <String>]
 [-PassThru] [-AsJob] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESCHREIBUNG
Wenn cloud tiering auf einem Serverendpunkt (einem bestimmten Speicherort auf einem registrierten Server) aktiviert ist, kann dieser Befehl verwendet werden, um alle mehrstufigen Dateien auf den lokalen Datenträger zurückrufen. Es wird dringend empfohlen, die Cloud tiering auf diesem Serverendpunkt zu deaktivieren, bevor Sie mit dem Rückruf beginnen. Wenn die Gestuftierung weiterhin besteht, kann der Rückruf dazu führen, dass andere Dateien gestuft werden, wodurch das gewünschte Ziel aller Inhalte, die sich auf dem lokalen Datenträger aufhalten, nicht erreicht wird.

## BEISPIELE

### Beispiel 1
```powershell
PS C:\> Invoke-AzStorageSyncFileRecall -ResourceGroupName "myResourceGroup" -StorageSyncServiceName "myStorageSyncServiceName" -SyncGroupName "mySyncGroupName" -ServerEndpointName "myServerEndpointName"
```

Dieser Befehl ruft alle mehrstufigen Dateien, die sich unter dem Stammpfad des angegebenen Serverendpunkts befinden, rekursiv zurück.

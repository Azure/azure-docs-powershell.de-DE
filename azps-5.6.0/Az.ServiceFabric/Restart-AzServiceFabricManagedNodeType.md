# Restart-AzServiceFabricManagedNodeType

## SYNOPSIS
Starten Sie bestimmte Knoten vom Knotentyp neu.

## SYNTAX

```
Restart-AzServiceFabricManagedNodeType [-ResourceGroupName] <String> [-ClusterName] <String> [-Name] <String>
 -NodeName <String[]> [-ForceRestart] [-PassThru] [-AsJob] [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

## BESCHREIBUNG
Starten Sie bestimmte Knoten vom Knotentyp neu. Es deaktiviert die Dienst-Fabric-Knoten, bevor die vms neu gestartet werden, und aktiviert sie wieder, sobald sie wiederkommen. Wenn dies für primäre Knotentypen erfolgt, kann es eine Weile dauern, da möglicherweise nicht alle Knoten gleichzeitig neu gestartet werden. Verwenden Sie -ForceRestart erzwingen den Vorgang, auch wenn service fabric die Knoten nicht deaktivieren kann, aber mit Vorsicht vorgehen kann, da dies zu Datenverlust führen kann, wenn auf dem Knoten zustandsreiche Arbeitsauslastungen ausgeführt werden.

## BEISPIELE

### Beispiel 1
```powershell
$rgName = "testRG"
$clusterName = "testCluster"
$NodeTypeName = "nt1"
Restart-AzServiceFabricManagedNodeType -ResourceGroupName $rgName -ClusterName $clusterName  -Name $NodeTypeName -NodeName nt1_0, nt1_3
```

Starten Sie knoten 0 und 3 für den Knotentyp neu.

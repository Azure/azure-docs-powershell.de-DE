# Add-AzWebAppTrafficRouting

## SYNOPSIS
Fügen Sie dem Slot eine Routingregel hinzu.

## SYNTAX

```
Add-AzWebAppTrafficRouting -ResourceGroupName <String> -WebAppName <String> -RoutingRule <Hashtable>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESCHREIBUNG
Das **Add-AzWebAppTrafficRouting-Cmdlet** fügt einem Azure Web App-Slot eine Routingregel hinzu.

## BEISPIELE

### Beispiel 1 Hinzufügen einer Routingregel zum Übertragen von 15 % des Produktionsdatenverkehrs an den Steckplatz "Stg"
```powershell
PS C:\>Add-AzWebAppTrafficRouting -ResourceGroupName "Default-Web-WestUS" -WebAppName "ContosoSite" 
-RoutingRule @{ActionHostName='XXXX.azurewebsites.net';ReroutePercentage=15;Name='Stg'}
```

Mit diesem Befehl wird eine Routingregel zum Übertragen von 15 % des Produktionsdatenverkehrs an den Steckplatz "Stg" addiert.

### Beispiel 2 Fügen Sie eine Routingregel hinzu, um den Produktionsdatenverkehr in inkrementelle Weise zwischen 50 % und 90 % zu übertragen.
```powershell
PS C:\>Add-AzWebAppTrafficRouting -ResourceGroupName "Default-Web-WestUS" -WebAppName "ContosoSite" 
-RoutingRule @{ActionHostName='XXXX.azurewebsites.net';ReroutePercentage=50;ChangeIntervalInMinutes=1;
MinReroutePercentage=50;MaxReroutePercentage=90;Name='Stg';ChangeStep=10}
```

Dieser Befehl fügt eine Routingregel hinzu, um den Produktionsdatenverkehr in inkrementelle Weise von 50 % bis 90 % an die Stg-Slotbereiche zu übertragen.

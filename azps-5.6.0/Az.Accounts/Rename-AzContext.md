# Rename-AzContext

## SYNOPSIS
Benennen Sie einen Azure-Kontext um.  Standardmäßig werden Kontexte nach Benutzerkonto und Abonnement benannt.

## SYNTAX

### RenameByInputObject (Standard)
```
Rename-AzContext -InputObject <PSAzureContext> [-Force] [-PassThru] [-Scope <ContextModificationScope>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [-TargetName] <String> [<CommonParameters>]
```

### RenameByName
```
Rename-AzContext [-Force] [-PassThru] [-Scope <ContextModificationScope>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [-SourceName] <String> [-TargetName] <String>
 [<CommonParameters>]
```

## BESCHREIBUNG
Benennen Sie einen Azure-Kontext um.  Standardmäßig werden Kontexte nach Benutzerkonto und Abonnement benannt.

## BEISPIELE

### Beispiel 1: Umbenennen eines Kontexts mit benannten Parametern
```powershell
PS C:\> Rename-AzContext -SourceName "[user1@contoso.org; 12345-6789-2345-3567890]" -TargetName "Work"
```

Benennen Sie den Kontext für user1@contoso.org " mit Abonnement '12345-6789-2345-3567890' in "Arbeit" um.  Nach diesem Befehl können Sie den Kontext mithilfe von "Select-AzContext Work" als Ziel verwenden.  Beachten Sie, dass Sie die Werte für "SourceName" mithilfe des Tabstoppabschlusses tabstoppen können.

### Beispiel 2: Umbenennen eines Kontexts mithilfe von Positionsparametern
```powershell
PS C:\> Rename-AzContext "My context" "Work"
```

Benennen Sie den Kontext mit dem Namen "Mein Kontext" in "Arbeit" um.  Nach diesem Befehl können Sie den Kontext mithilfe Select-AzContext

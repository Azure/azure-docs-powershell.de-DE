# Set-AzDataLakeStoreItemPermission

## SYNOPSIS
Ändert die Oktalberechtigung einer Datei oder eines Ordners im Data Lake Store.

## SYNTAX

```
Set-AzDataLakeStoreItemPermission [-Account] <String> [-Path] <DataLakeStorePathInstance> [-Permission] <Int32>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESCHREIBUNG
Das **Cmdlet Set-AzDataLakeStoreItemPermission** ändert das Berechtigungs oktal einer Datei oder eines Ordners im Data Lake Store.

## BEISPIELE

### Beispiel 1: Festlegen der Oktalberechtigung für ein Element
```
PS C:\>Set-AzDataLakeStoreItemPermission -AccountName "ContosoADL" -Path "/file.txt" -Permission 0770
```

Dieser Befehl legt die Oktalberechtigung für eine Datei auf 0770 fest, was dazu führt, dass das Sticky-Bit, das Festlegen von Lese-/Schreib-/Ausführungsberechtigungen für den Besitzer der Datei, das Festlegen von Lese-/Schreib-/Ausführungsberechtigungen für die besitzende Gruppe der Datei und das Löschen von Lese-/Schreib-/Ausführungsberechtigungen für andere personen.

# Remove-AzIntegrationAccountReceivedIcn

## SYNOPSIS
Dieses Cmdlet entfernt eine bestimmte empfangene Austauschkontrollnummer pro Vertrag und Steuerelementnummerwert.

## SYNTAX

```
Remove-AzIntegrationAccountReceivedIcn -ResourceGroupName <String> -Name <String> -AgreementName <String>
 -ControlNumberValue <String> [-AgreementType <String>] [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

## BESCHREIBUNG
Dieses Cmdlet soll in Notfallwiederherstellungsszenarien verwendet werden, um eine empfangene Austauschsteuerelementnummer aus dem Integrationskonto zu entfernen, damit der B2B-Verbinder die Nachricht erneut verarbeiten kann, wenn die Erkennung doppelter Nummern aktiviert ist.
In seltenen Fällen kann die empfangene Austauschsteuerelementnummer kurz vor einem Notfall reserviert werden, bevor der B2B-Verbinder den Austausch als fehlerhaft ablehnt.
In solchen Fällen kann der Vorgang der Wiederherstellungswebsite ermöglichen, denselben Austausch erneut zu verarbeiten, nachdem die Nutzlast korrigiert wurde.
Geben Sie den Parameter "-AgreementType" an, um anzugeben, ob X12- oder Edifact-Steuerelementnummern

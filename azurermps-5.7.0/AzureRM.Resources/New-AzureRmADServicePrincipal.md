# New-AzureRmADServicePrincipal

## Synopsis
Erstellt einen neuen Azure Active Directory-Dienstprinzipal.

[!INCLUDE [migrate-to-az-banner](../../includes/migrate-to-az-banner.md)]

## Syntax

New-AzureRmADServicePrincipal -ApplicationId <Guid> [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]

## Beschreibung
Erstellt einen neuen Azure Active Directory-Dienstprinzipal.

Hinweis: das Cmdlet erstellt auch implizit eine Anwendung und legt deren Eigenschaften fest (wenn die ApplicationId nicht bereitgestellt wird).
Verwenden Sie Set-AzureRmADApplication-Cmdlet, um die anwendungsspezifischen Parameter zu aktualisieren.

## Beispiele

### Beispiel 1
```
New-AzureRmADServicePrincipal -ApplicationId 34a28ad2-dec4-4a41-bc3b-d22ddf90000e
```

Erstellt einen neuen Azure Active Directory-Dienstprinzipal.

DisplayName-Typ-ObjectID
-----------                    ----                           --------
DemoApp ServicePrincipal f95b6f5c-fc98-4af0-bb8a-34a14ca1dca1

### Beispiel 2
```
$SecureStringPassword = ConvertTo-SecureString -String "password" -AsPlainText -Force
New-AzureRmADServicePrincipal -DisplayName SPForNoExistingApp -Password $SecureStringPassword
```

Erstellt einen neuen Dienstprinzipal.
Das Cmdlet erstellt auch implizit eine Anwendung, da eine Anwendung nicht bereitgestellt wird.

DisplayName-Typ-ObjectID
-----------                    ----                           --------
SPForNoExistingApp ServicePrincipal 784136ca-3ae2-4fdd-a388-89d993e7c780

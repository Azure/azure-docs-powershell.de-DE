# New-WAPackStaticIPAddressPool

## Synopsis
Erstellt einen statischen IP-Adresspool.

## Syntax

```
New-WAPackStaticIPAddressPool -VMSubnet <VMSubnet> -Name <String> -IPAddressRangeStart <String>
 -IPAddressRangeEnd <String> [-Profile <AzureSMProfile>] [<CommonParameters>]
```

## Beschreibung
Diese Themen sind veraltet und werden in Zukunft entfernt.
In diesem Thema wird das Cmdlet in der Version 0.8.1 des Microsoft Azure PowerShell-Moduls beschrieben.
Wenn Sie die Version des verwendeten Moduls ermitteln möchten, geben Sie aus der Azure PowerShell-Konsole ein `(Get-Module -Name Azure).Version` .

Mit dem Cmdlet **New-WAPackStaticIPAddressPool** wird ein statischer IP-Adresspool erstellt.

## Beispiele

### Beispiel 1: Erstellen eines statischen IP-Adresspools
```
PS C:\> $VNet = Get-WAPackVNet -Name "ContosoVNet01"
PS C:\> $VMSubnet = Get-WAPackVMSubnet -VNet $VNet -Name "ContosoVMSubnet01"
PS C:\> New-WAPackStaticIpAddressPool ?VMSubnet $VMSubnet -Name "ContosoStaticIpAddressPool01" -IPAddressRangeStart "192.168.1.0" -IPAddressRangeEnd "192.168.1.10"
```

Der erste Befehl ruft zuerst das virtuelle Computernetzwerk ab, dem der statische IP-Adresspool hinzugefügt werden soll.
Dieses Virtual Machine-Netzwerk hat den Namen ContosoVNet01.

Der zweite Befehl verwendet das zuvor abgerufene Virtual Machine-Netzwerk, um das Subnetz des virtuellen Computers mit dem Namen ContosoVMSubnet01 abzurufen, dem der statische IP-Adresspool hinzugefügt werden soll.

Mit dem letzten Befehl wird ein neuer statischer IP-Adresspool mit einem Namen ContosoStaticIpAddressPool01 und einem Bereichs Start-192.168.1.0 und einem Bereichsende-192.168.1.10 erstellt.

# Wait-AzureHDInsightJob

## Synopsis
Erwartet den Abschluss oder Fehler eines HDInsight-Auftrags und zeigt den Status des Auftrags an.

## Syntax

### Abrufen des Tabelle Jobdetails-Verlaufs eines HDInsight-Clusters (Standard)
```
Wait-AzureHDInsightJob [-Credential <PSCredential>] [-WaitTimeoutInSeconds <Double>]
 [-Profile <AzureSMProfile>] [<CommonParameters>]
```

### Abrufen des Tabelle Jobdetails-Verlaufs eines HDInsight-Clusters (mit bestimmten Anmeldeinformationen für Abonnements)
```
Wait-AzureHDInsightJob [-Certificate <X509Certificate2>] [-HostedService <String>] [-Endpoint <Uri>]
 [-IgnoreSslErrors <Boolean>] -Job <AzureHDInsightJob> -Subscription <String> [-WaitTimeoutInSeconds <Double>]
 [-Profile <AzureSMProfile>] [<CommonParameters>]
```

### Warten des Auftrags mit JobID in einem HDInsight-Cluster
```
Wait-AzureHDInsightJob -Cluster <String> [-Credential <PSCredential>] -JobId <String>
 [-WaitTimeoutInSeconds <Double>] [-Profile <AzureSMProfile>] [<CommonParameters>]
```

### Job mit Job auf einem HDInsight-Cluster warten
```
Wait-AzureHDInsightJob [-Credential <PSCredential>] -Job <AzureHDInsightJob> [-WaitTimeoutInSeconds <Double>]
 [-Profile <AzureSMProfile>] [<CommonParameters>]
```

Das Cmdlet **Wait-AzureHDInsightJob** wartet auf den Abschluss oder Fehler eines Azure HDInsight-Auftrags und zeigt den Fortschritt des Auftrags an.

## Beispiele

### Beispiel 1: Ausführen eines Auftrags und abwarten des Abschlusses
```
PS C:\>$SubId = (Get-AzureSubscription -Current).SubscriptionId
PS C:>\ $ClusterName = "MyCluster"
PS C:>\ $WordCountJob = New-AzureHDInsightMapReduceJobDefinition -JarFile "/Example/Apps/Hadoop-examples.jar" -ClassName "Wordcount" -Defines @{ "mapred.map.tasks" = "3" } -Arguments "/Example/Data/Gutenberg/Davinci.txt", "/Example/Output/WordCount" 
PS C:>\ $WordCountJob | Start-AzureHDInsightJob -Subscription $SubId -Cluster $ClusterName 
    | Wait-AzureHDInsightJob -Subscription $SubId -WaitTimeoutInSeconds 3600 
    | Get-AzureHDInsightJobOutput -Cluster $ClusterName -Subscription $SubId -StandardError
```

Der erste Befehl ruft die aktuelle Azure-Abonnement-ID ab und speichert Sie dann in der $SubId-Variablen.

Der zweite Befehl ruft den angegebenen Cluster ab und speichert ihn dann in der $Clustername-Variablen.

Der dritte Befehl verwendet das Cmdlet **New-AzureHDInsightMapReduceJobDefinition** , um eine MapReduce-Auftragsdefinition zu erstellen, und speichert Sie dann in der $WordCountJob-Variablen.

Der vierte Befehl verwendet mehrere Cmdlets in der folgenden Reihenfolge: 

- Er verwendet den Pipelineoperator, um $WordCountJob an das **Start-AzureHDInsightJob-** Cmdlet zu übergeben, um den Auftrag zu starten. 
- Der Auftrag wird an das **Wait-AzureHDInsightJob-** Cmdlet übergeben, um 3600 Sekunden für den Abschluss des Auftrags zu warten. 
- Wenn der Auftrag abgeschlossen ist, verwendet der Befehl das Cmdlet " **Get-AzureHDInsightJobOutput** ", um die Ausgabe des Auftrags abzurufen.

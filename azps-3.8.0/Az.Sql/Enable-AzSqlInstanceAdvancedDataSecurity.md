# Enable-AzSqlInstanceAdvancedDataSecurity

## Synopsis
Ermöglicht erweiterte Datensicherheit für eine verwaltete Instanz.

## Syntax

```
Enable-AzSqlInstanceAdvancedDataSecurity [-DoNotConfigureVulnerabilityAssessment] [-AsJob]
 [-DeploymentName <String>] [-InputObject <AzureSqlManagedInstanceModel>] -InstanceName <String>
 [-ResourceGroupName] <String> [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## Beschreibung
Das Cmdlet **enable-AzSqlInstanceAdvancedDataSecurity** ermöglicht erweiterte Datensicherheit für eine verwaltete Instanz. Advanced Data Security ist ein einheitliches Sicherheitspaket, das die Datenklassifizierung, die Anfälligkeitsbewertung und den erweiterten Bedrohungsschutz für Ihre verwaltete Instanz umfasst. (Es wird automatisch ein neues Speicherkonto zum Speichern von Sicherheitsrisikobewertungen erstellt. Wenn zuvor ein Speicherkonto für diesen Zweck erstellt wurde, wird es stattdessen verwendet.

## Beispiele

### Beispiel 1 – erweiterte Datensicherheit für verwaltete Instanzen aktivieren
```powershell
PS C:\>  Enable-AzSqlInstanceAdvancedDataSecurity `
            -ResourceGroupName "ResourceGroup01" `
            -InstanceName "ManagedInstance01" 

ResourceGroupName            : ResourceGroup01
ManagedInstanceName          : ManagedInstance01
IsEnabled                    : True
```

### Beispiel 2 – erweiterte Datensicherheit für verwaltete Instanzen von Serverressourcen aktivieren
```powershell
PS C:\>  Get-AzSqlInstance `
           -ResourceGroupName "ResourceGroup01" `
           -Name "ManagedInstance01" `
           | Enable-AzSqlInstanceAdvancedDataSecurity

ResourceGroupName            : ResourceGroup01
ManagedInstanceName          : ManagedInstance01
IsEnabled                    : True
```

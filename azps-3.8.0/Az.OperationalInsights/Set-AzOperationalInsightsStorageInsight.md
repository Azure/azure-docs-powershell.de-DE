# Set-AzOperationalInsightsStorageInsight

## Synopsis
Aktualisiert eine Speicher-Insight.

## Syntax

### ByWorkspaceName (Standard)
```
Set-AzOperationalInsightsStorageInsight [-ResourceGroupName] <String> [-WorkspaceName] <String>
 [-Name] <String> [[-StorageAccountKey] <String>] [[-Tables] <String[]>] [[-Containers] <String[]>]
 [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

### ByWorkspaceObject
```
Set-AzOperationalInsightsStorageInsight [-Workspace] <PSWorkspace> [-Name] <String>
 [[-StorageAccountKey] <String>] [[-Tables] <String[]>] [[-Containers] <String[]>]
 [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

## Beschreibung
Das Cmdlet " **Satz-AzOperationalInsightsStorageInsight** " ändert die Konfiguration einer Speicher-Insight.

## Beispiele

### Beispiel 1: Ändern eines Speicher Einblicks anhand des Namens
```
PS C:\>Set-AzOperationalInsightsStorageInsight -ResourceGroupName "ContosoResourceGroup" -WorkspaceName "MyWorkspace" -Name "MyStorageInsight" -Tables @("WADWindowsEventLogsTable")
```

Mit diesem Befehl werden die Tabellen geändert, aus denen die Speicher Insight mit dem Namen MyStorageInsight liest.

### Beispiel 2: Ändern eines Speicher Einblicks mithilfe eines Workspace-Objekts
```
PS C:\>$Workspace = Get-AzOperationalInsightsWorkspace -ResourceGroupName "ContosoResourceGroup" -Name "MyWorkspace"

PS C:\>Set-AzOperationalInsightsStorageInsight -Workspace $Workspace -Name "MyStorageInsight" -Containers @("wad-iis-logfiles")
```

Der erste Befehl verwendet das Get-AzOperationalInsightsWorkspace-Cmdlet, um den Arbeitsbereich mit dem Namen myworkspace abzurufen, und speichert ihn dann in der Variablen $Workspace.
Mit dem zweiten Befehl werden die Container geändert, aus denen die Speicher Insight mit dem Namen MyStorageInsight liest.

# New-AzDataFactoryGatewayAuthKey

## Synopsis
Erstellt den auth-Schlüssel für ein Azure Data Factory-Gateway.

## Syntax

### ByFactoryName (Standard)
```
New-AzDataFactoryGatewayAuthKey [-DataFactoryName] <String> [-GatewayName] <String> [-KeyName] <String>
 [-ResourceGroupName] <String> [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

### ByFactoryObject
```
New-AzDataFactoryGatewayAuthKey [-InputObject] <PSDataFactory> [-GatewayName] <String> [-KeyName] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## Beschreibung
Das Cmdlet **New-AzDataFactoryGatewayAuthKey** erstellt Gateway-auth-Schlüssel für ein angegebenes Azure Data Factory-Gateway.
Mit diesem Schlüssel registrieren Sie das Gateway bei einem clouddienst.

## Beispiele

### Beispiel 1: erstellt einen Gateway-auth-Schlüssel für key1
```
PS C:\> New-AzDataFactoryGatewayAuthKey -ResourceGroup ADFResource -GatewayName 'MyGateway' -DataFactoryName MyADF -KeyName key1
Key1 : DMG@632e739e-1053-4070-9102-8591f067526e@41fcbc45-c594-4152-a8f1-fcbcd6452aea@wu@BH0EV9hu/o2IYGQzfYYD203XhdS6Tty
       fkYwYFbG6wBU=
Key2 :
```

Dieser Befehl erstellt einen Gateway-auth-Schlüssel von key1 für das Data Factory-Gateway mit dem Namen mygateway.

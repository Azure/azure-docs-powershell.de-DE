# Get-AzTrafficManagerEndpoint

## SYNOPSIS
Ruft einen Endpunkt für ein Datenverkehrs-Manager-Profil ab.

## SYNTAX

### Felder
```
Get-AzTrafficManagerEndpoint -Name <String> -Type <String> -ProfileName <String> -ResourceGroupName <String>
 [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

### Objekt
```
Get-AzTrafficManagerEndpoint -TrafficManagerEndpoint <TrafficManagerEndpoint>
 [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

## BESCHREIBUNG
Das **Cmdlet "Get-AzTrafficManagerEndpoint"** ruft einen Endpunkt für ein Azure Traffic Manager-Profil ab.

Sie können dieses Objekt lokal ändern und dann Änderungen am Profil mithilfe des Set-AzTrafficManagerEndpoint anwenden.
Geben Sie den Endpunkt mithilfe der *Parameter "Name"* und *"Type"* an.
Sie können das Datenverkehrs-Manager-Profil entweder mithilfe des Parameters *"ProfileName"* und *"ResourceGroupName"* oder durch Angeben eines **"TrafficManagerProfile"-Objekts** angeben.
Alternativ können Sie den Wert mithilfe der Pipeline übergeben.

## BEISPIELE

### Beispiel 1: Erhalten eines Endpunkts
```
PS C:\>$TrafficManagerEndpoint = Get-AzTrafficManagerEndpoint -Name "contoso" -ProfileName "ContosoProfile" -ResourceGroupName "ResourceGroup11" -Type AzureEndpoints
```

Dieser Befehl ruft den Azure-Endpunkt namens "contoso" aus dem Profil "ContosoProfile" in der Ressourcengruppe "ResourceGroup11" ab und speichert dieses Objekt dann in der $TrafficManagerEndpoint Variable.

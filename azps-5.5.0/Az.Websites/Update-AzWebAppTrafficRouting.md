# Update-AzWebAppTrafficRouting

## SYNOPSIS
Aktualisieren Sie eine Routingregel für das Slot.

## SYNTAX

```
Update-AzWebAppTrafficRouting -ResourceGroupName <String> -WebAppName <String> -RoutingRule <Hashtable>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESCHREIBUNG
Das **Cmdlet "Update-AzWebAppTrafficRouting"** aktualisiert die Routingregelkonfiguration für ein Azure Web App-Slot.

## BEISPIELE

### Beispiel 1 Aktualisieren einer Routingregel, um 15 % des Produktionsdatenverkehrs an das Slot "Stg" zu übertragen
```powershell
PS C:\>Update-AzWebAppTrafficRouting -ResourceGroupName "Default-Web-WestUS" -WebAppName "ContosoSite" 
- RoutingRule @{AtionHostName='XXXX.azurewebsites.net';ReroutePercentage=15;Name='Stg'}
```

Mit diesem Befehl wird eine Routingregel aktualisiert, um 15 % des Produktionsdatenverkehrs an das Slot "Stg" zu übertragen.

### Beispiel 2 Aktualisieren Sie eine Routingregel, um den Produktionsdatenverkehr inkrementell zwischen 50 % und 90 % zu übertragen.
```powershell
PS C:\>Update-AzWebAppTrafficRouting -ResourceGroupName "Default-Web-WestUS" -WebAppName "ContosoSite" 
-RoutingRule @{ActionHostName='XXXX.azurewebsites.net';ReroutePercentage=50;ChangeIntervalInMinutes=1;
MinReroutePercentage=50;MaxReroutePercentage=90;Name='Stg';ChangeStep=10}
```

Mit diesem Befehl wird eine Routingregel aktualisiert, um den Produktionsdatenverkehr inkrementell von 50 % bis 90 % zu übertragen.

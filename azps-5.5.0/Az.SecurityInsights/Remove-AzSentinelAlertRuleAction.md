# Remove-AzSentinelAlertRuleAction

## SYNOPSIS
Entfernen einer automatisierten Antwort aus einer Analyse

## SYNTAX

### ActionId (Standard)
```
Remove-AzSentinelAlertRuleAction -ResourceGroupName <String> -WorkspaceName <String> -AlertRuleId <String>
 -ActionId <String> [-PassThru] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

### InputObject
```
Remove-AzSentinelAlertRuleAction -InputObject <PSSentinelActionResponse> [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESCHREIBUNG
Das **Cmdlet "Remove-AzSentinelAlertRuleAction"** löscht eine automatisierte Antwort aus der Warnungsregel in einem angegebenen Arbeitsbereich endgültig.
Sie können ein **AlertRuleAction-Objekt** mit dem Pipelineoperator übergeben, oder Sie können die erforderlichen Parameter angeben.
Sie können den Parameter "Bestätigen" und $ConfirmPreference Windows PowerShell, um zu steuern, ob sie vom Cmdlet zur Bestätigung aufgefordert werden.

## BEISPIELE

### Beispiel 1
```powershell
PS C:\> Remove-AzSentinelAlertRuleAction -ResourceGroupName "MyResourceGroup" -WorkspaceName "MyWorkspaceName" -AlertRuleId "MyAlertRuleId" -ActionId "MyActionId"
```

Mit diesem Befehl wird die Warnungsregel aus dem Arbeitsbereich entfernt.

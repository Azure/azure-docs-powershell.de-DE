# New-AzDataMigrationService

## SYNOPSIS
Erstellt eine neue Instanz des Azure-Datenbankmigrationsdiensts.

## SYNTAX

```
New-AzDataMigrationService -ResourceGroupName <String> -Name <String> -Location <String> -Sku <String>
 -VirtualSubnetId <String> [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## BESCHREIBUNG
Das New-AzDataMigrationService erstellt eine neue Instanz des Azure-Datenbankmigrationsdiensts. Dieses Cmdlet trägt den Namen der vorhandenen Azure-Ressourcengruppe, den eindeutigen Namen für die neue Instanz des zu erstellenden Azure-Datenbankmigrationsdiensts, die Region, in der die Instanz bereitgestellt wird, den Namen der DMS-Worker-SKU und den Namen des virtuellen Azure-Subnetzes, in dem sich der Dienst befinden soll. Es gibt keinen Parameter für den Abonnementnamen, da erwartet wird, dass der Benutzer das Standardabonnement der Azure-Anmeldesitzung anbeschriften oder Get-AzSubscription -SubscriptionName "MySubscription" | Select-AzSubscription, um ein anderes Abonnement auszuwählen.

## BEISPIELE

### Beispiel 1
```
PS C:\> New-AzDataMigrationService -ResourceGroupName myResourceGroup -Name TestService -Location "Central US" -Sku Basic_2vCores -VirtualSubnetId $virtualSubNetId
```

Das beispiel oben zeigt, wie Sie eine neue Instanz des Azure Database Migration Service namens TestService in der Region "USA, Mitte" erstellen.

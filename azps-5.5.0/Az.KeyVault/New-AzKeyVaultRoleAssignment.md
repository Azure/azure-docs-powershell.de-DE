# New-AzKeyVaultRoleAssignment

## SYNOPSIS
Weist dem angegebenen Prinzipal im angegebenen Bereich die angegebene RbAC-Rolle zu.

## BESCHREIBUNG
Verwenden Sie den `New-AzKeyVaultRoleAssignment` Befehl, um den Zugriff zu gewähren.
Access wird gewährt, indem ihnen im richtigen Bereich die entsprechende RbAC-Rolle zugewiesen wird.
Der Betreff der Zuordnung muss angegeben werden.
Verwenden Sie zum Angeben eines Benutzers die Parameter "SignInName" oder "Azure AD ObjectId".
Verwenden Sie den Azure AD ObjectId-Parameter, um eine Sicherheitsgruppe anzugeben.
Verwenden Sie zum Angeben einer Azure AD-Anwendung die Parameter ApplicationId oder ObjectId.
Die zugewiesene Rolle muss mit dem Parameter "RoleDefinitionName pr RoleDefinitionId" angegeben werden. Der Bereich, für den der Zugriff gewährt wird, kann angegeben werden. Standardmäßig wird das ausgewählte Abonnement verwendet.

## BEISPIELE

### Beispiel 1
```powershell
PS C:\> New-AzKeyVaultRoleAssignment -HsmName myHsm -RoleDefinitionName "Managed HSM Policy Administrator" -SignInName user1@microsoft.com

RoleDefinitionName               DisplayName                    ObjectType Scope
------------------               -----------                    ---------- -----
Managed HSM Policy Administrator User 1 (user1@microsoft.com)   User       /
```

In diesem Beispiel wird dem Benutzer "" im oberen Bereich die Rolle "Verwalteter user1@microsoft.com HSM-Richtlinienadministrator" zugewiesen.

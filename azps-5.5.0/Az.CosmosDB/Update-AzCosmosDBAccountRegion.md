# Update-AzCosmosDBAccountRegion

## SYNOPSIS
Aktualisieren von Regionen eines Kontos vom 8.

## SYNTAX

### ByNameParameterSet (Standard)
```
Update-AzCosmosDBAccountRegion -ResourceGroupName <String> -Name <String> [-Location <String[]>]
 [-LocationObject <PSLocation[]>] [-AsJob] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

### ByResourceIdParameterSet
```
Update-AzCosmosDBAccountRegion [-Location <String[]>] [-LocationObject <PSLocation[]>] -ResourceId <String>
 [-AsJob] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ByObjectParameterSet
```
Update-AzCosmosDBAccountRegion [-Location <String[]>] [-LocationObject <PSLocation[]>]
 -InputObject <PSDatabaseAccountGetResults> [-AsJob] [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

## BESCHREIBUNG
Aktualisieren von Regionen eines Kontos vom 8. Der Speicherort kann entweder als Objekt vom Typ "PSLocation" oder als nach Failoverpriorität geordnete Zeichenfolge "Location Name" bereitgestellt werden.
Der Parameter "LocationObject" erwartet, dass die Liste der aktuellen Speicherorte (eingeschlossene Failover priorisierungen) von den neuen "LocationObjects" angefügt wird, die neuen Speicherorten entsprechend hinzugefügt werden.
Der Parameter "Location" erwartet die Liste der aktuellen Position (nach Failoverpriorität geordnet) und die neuen Speicherorte. Beachten Sie, dass wir nur das Addition von Regionen unterstützen. Geben Sie entweder "Location" oder "LocationObject" an.

# New-AzureRmSqlDatabaseCopy

## Synopsis
Erstellt eine Kopie einer SQL-Datenbank, die den Snapshot zum aktuellen Zeitpunkt verwendet.

[!INCLUDE [migrate-to-az-banner](../../includes/migrate-to-az-banner.md)]

## Syntax

### DtuBasedDatabase (Standard)
```
New-AzureRmSqlDatabaseCopy [-DatabaseName] <String> [-ServiceObjectiveName <String>]
 [-ElasticPoolName <String>] [-Tags <Hashtable>] [-CopyResourceGroupName <String>] [-CopyServerName <String>]
 -CopyDatabaseName <String> [-AsJob] [-LicenseType <String>] [-ServerName] <String>
 [-ResourceGroupName] <String> [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

### VcoreBasedDatabase
```
New-AzureRmSqlDatabaseCopy [-DatabaseName] <String> [-Tags <Hashtable>] [-CopyResourceGroupName <String>]
 [-CopyServerName <String>] -CopyDatabaseName <String> [-AsJob] -ComputeGeneration <String> -VCore <Int32>
 [-LicenseType <String>] [-ServerName] <String> [-ResourceGroupName] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## Beschreibung
Das Cmdlet **New-AzureRmSqlDatabaseCopy** erstellt eine Kopie einer Azure SQL-Datenbank, in der die Momentaufnahme der Daten zur aktuellen Zeit verwendet wird. Verwenden Sie dieses Cmdlet anstelle des Start-AzureSqlDatabaseCopy-Cmdlets, um eine einmalige Datenbankkopie zu erstellen. Dieses Cmdlet gibt das **Daten Bank** Objekt der Kopie zurück.
Hinweis: Verwenden Sie das New-AzureRmSqlDatabaseSecondary-Cmdlet, um die Geo-Replikation für eine Datenbank zu konfigurieren.
Dieses Cmdlet wird auch vom SQL Server Stretch-Datenbankdienst auf Azure unterstützt.

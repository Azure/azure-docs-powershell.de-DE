# Resolve-AzureRmError

## Synopsis
Zeigen Sie detaillierte Informationen zu PowerShell-Fehlern mit erweiterten Details für Azure PowerShell-Fehler an.

[!INCLUDE [migrate-to-az-banner](../../includes/migrate-to-az-banner.md)]

## Syntax

### AnyErrorParameterSet (Standard)
```
Resolve-AzureRmError [[-Error] <ErrorRecord[]>] [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

### LastErrorParameterSet
```
Resolve-AzureRmError [-Last] [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

## Beschreibung
Behebt und zeigt detaillierte Informationen zu Fehlern in der aktuellen PowerShell-Sitzung an, einschließlich der Fehler im Skript, in der Stapelüberwachung sowie in allen inneren und Aggregat Ausnahmen. Für Azure PowerShell-Fehler bietet zusätzliche Informationen zum Debuggen von Dienstproblemen, einschließlich vollständiger Details zur Anforderung und Serverantwort, die den Fehler verursacht haben.

```yaml
Position: Named
Default value: None
Accept pipeline input: False
Accept wildcard characters: False
```

### -Fehler
Eine oder mehrere Fehlereinträge, die aufgelöst werden sollen.  Wenn keine Parameter angegeben werden, werden alle Fehler in der Sitzung aufgelöst.

```yaml
Position: 0
Default value: None
Accept pipeline input: True (ByValue)
Accept wildcard characters: False
```
